Spring AOP学习

Posted on 2018-07-16 | In Note

基本概念

AOP 框架其实可以将它理解为一个拦截器框架，如果它拦截了一个类，那么它就拦截了这个类中所有的方法。在项目中往往只需要拦截特定的方法就行了，为了解决所有方法被拦截这个问题，引入了切面的概念。

| 概念 | 作用 |
| 切点（PointCut） | 在哪个地方进行切入,它可以指定某一个点，也可以指定多个点 |
| 通知（advice） | 在切入点干什么，切面的实际实现，打日志、执行缓存、处理异常等等 |
| 切面（aspect） | PointCut + Advice 形成了切面Aspect要实现的交叉功能，是系统模块化的一个切面或者领域，如日志记录 |
| 代理 | 将通知用到目标对象后创建的对象，应用系统的其他部分不用为支持代理对象而改变 |
| 目标对象（target） | 被通知的对象 |
| 织入 | 将切面应用到目标对象从而创建一个新代理对象的过程，织入发生在目标对象生命周期的多个点上。在配置interceptionForNames时发生 |
| 引入 | 为类添加新方法和属性 |

当通过代理对象去实现aop时，获取的ProxyFacotoryBean是什么类型？
返回的是一个代理对象，如果目标对象实现了接口，则spring使用jdk动态代理技术；如果没有实现接口，则使用CGLib技术。

代理

静态和动态是由代理产生的时间段来决定的。静态代理产生于代码编译阶段，动态代理产生于运行阶段。

我们以一个例子来理解代理：打日志

静态代理

这种代理方式需要编写固定的接口，并且需要实现接口中的每一个方法。

动态代理

我们不想实现接口中的每一个方法，希望能够编写代理作用于所有的函数上。这样可以通过运行期的反射机制实现。

这种做法有一个局限，只有实现了接口的类才能为其实现代理。

CGLib

CGLib在运行时期（非编译时期）生成被代理对象的子类，并重写了被代理对象的所有方法，从而作为代理对象。
相当于CGLib帮我们完成了静态代理。
（注意：final类无法生成子类，所以无法使用CGLib）
