HTTP 406 not acceptable错误学习

Posted on 2019-03-13

从问题入门

问题描述

想为一个thrift服务添加http接口，实际上这个服务已经通过bootstrap程序启动了jetty容器，jetty端口也起来了，所以只用配置一些spring mvc相关内容就可以了。但是在添加了一个controller后，发现调用接口前端显示 HTTP 406 not acceptable

原因

406一般原因是客户端请求的数据类型和从服务端接收来的数据的类型不一致


从上图可以看出，状态码406
请求头（Request Headers）中的请求信息是json格式
响应头（Response Hraders）中却发现返回信息的格式是“text/html”

请求信息与响应信息不符合，前端无法解析，需将结果转换成json格式返回给前台。

解决

使用fastjson，引入jar包后配置，在spring-servlet.xml中加上配置即可

jackson加载简介

spring mvc配置了<mvc:annotation-driver/>后 ，如果 classpath 里面包含 jackson 包，则自动注册MappingJackson2HttpMessageConverter，从而支持json 输出。

Spring中是在AnnotationDrivenBeanDefinitionParser中解析<mvc:annotation-driver/>

通过源码可以看出，通过判断jackson包的ObjectMapper和JsonGenerator是否存在来确定jackson包被引入，被引入了就添加一个HttpMessageConverter

解析完成之后，在mvc启动时默认会加载下面的几种HttpMessageConverter，相关代码在 org.springframework.web.servlet.config.annotation.WebMvcConfigurationSupport中的addDefaultHttpMessageConverters方法中

如何影响前端展示

http的基础约定

SpringMVC启动时会自动配置一些HttpMessageConverter，接收到http请求时，从这些Converters中选择一个符合条件的来进行Http序列化/反序列化。

  1. 决定resp.body的Content-Type的第一要素是对应的req.headers.Accept属性的值，又叫做MediaType。如果服务端支持这个Accept，那么应该按照这个Accept来确定返回resp.body对应的格式，同时把resp.headers.Content-Type设置成自己支持的符合那个Accept的MediaType。服务端不支持Accept指定的任何MediaType时，应该返回错误406 Not Acceptable.
    例如：req.headers.Accept = text/html，服务端支持的话应该让resp.headers.Content-Type = text/html，并且resp.body按照html格式返回。
    例如：req.headers.Accept = text/asdfg，服务端不支持这种MediaType，应该返回406 Not Acceptable。
  2. 如果Accept指定了多个MediaType，并且服务端也支持多个MediaType，那么Accept应该同时指定各个MediaType的QualityValue，也就是q值，服务端根据q值的大小来决定这几个MediaType类型的优先级，一般是大的优先。q值不指定时，默认视为q=1.
    Chrome的默认请求的Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,image/apng,/;q=0.8，表示服务端在支持的情况下应该优先返回text/html，其次是application/xhtml+xml.
    前面几个都不支持时，服务器可以自行处理 /，返回一种服务器自己支持的格式。
  3. 一个HTTP请求没有指定Accept，默认视为指定 Accept: /；没有指定Content-Type，默认视为 null，就是没有。当然，服务端可以根据自己的需要改变默认值。
  4. Content-Type必须是具体确定的类型，不能包含 *.

RequestResponseBodyProcessor实现了HandlerMethodReturnValueHandler、HandlerMethodAgumentResolver接口，
所以具备了处理参数、处理返回值的能力。RequestResponseBodyProcessor能够解析用@RequestBody注解的参数和通过使用HttpMessageConverter读取并写入请求体或响应来处理用@ResponseBody注解的方法的返回值。

handleReturnValue()方法获取inputMessage,outMessage对象，调用writeWithMessageConverters()方法，让消息转换器对消息进行下一步处理
