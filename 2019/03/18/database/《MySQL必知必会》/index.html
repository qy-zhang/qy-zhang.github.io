Posted on 2019-03-18

SQL基础

语法

  • 多条SQL语句必须分号分隔
  • SQL语句不区分大小写，建议对所有SQL关键字使用大写，而对所有列和表名使用小写，这样做会使代码更易于阅读和调试
  • 在处理SQL语句时，其中的所有空格都被忽略。SQL语句可以在一行上给出，也可以分成许多行便于阅读和调试

主键

主键（primary key）一列（或一组列），其值能够唯一区分表中每一行

唯一标识表中每行的这一列（或这组列）成为主键。虽然并不总是都需要主键，但是建议每个表都具备一个主键，以便于以后的数据操纵和管理。

主键值规则

MySQL本身强制规定：

  • 任意两行都不具有相同的主键值
  • 每个行都必须具有一个主键值（主键列不允许NULL值）

主键的最好习惯

  • 不更新主键列中的值
  • 不重用主键列的值
  • 不在主键列中使用可能会更改的值。（例如如果使用供应商名字作为主键以标识某个供应商，当该供应商合并和更改其名字时，必须更改这个主键）

通配符过滤

通配符（wildcard）：用来匹配值得一部分特殊字符
搜索模式（search pattern）由字面值、通配符或两者组合构成的搜索条件

为了在搜索子句中使用通配符，必须使用LIKE操作符，LIKE指示MySQL，后跟的搜索模式利用通配符匹配而不是直接相等匹配进行比较

谓词（predicate）：判断是否存在满足某种条件的记录，如果存在返回TRUE，不存在返回FALSE。主要有：LIKE、BETWEEN、IS NULL、IS NOT NULL、IN、EXISTS

通配符搜索的处理一般要比前面讨论的其他搜索所花时间更长。

  • 不要过度使用通配符，如果其他操作符能够达到相同的目的，应该使用其他操作符
  • 尽量不要把通配符放到搜索的开始处，这样搜索起来最慢

更新和删除数据

更新UPDATE

UPDATE语句由三个部分组成：要更新的表、列名和他们的新值、确定要更新行的过滤条件。

UPDATE语句以WEHRE子句结束，它告诉MySQL更新哪一行。如果没有WHERE子句，MySQL将更新表中的所有行！

删除数据

  • DELETE 不需要列名或者通配符，他删除的是整行，如果需要删除指定的列中的内容，请考虑使用UPDATE语句
  • 不加WHERE子句将删除所有的行
  • 如果想从表中删除所有行，可以使用TRRUNCATE TABLE语句，他的速度更快（删除原来的表并重新创建一个表，而不是逐行删除）
