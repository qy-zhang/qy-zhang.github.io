MySQL查询区分大小写

Posted on 2019-05-21

介绍

我们知道使用 WHERE 来执行匹配是不区分大小写的，同时在使用 LIKE 谓词来进行模糊匹配的时候也没有区分大小写
解决此问题有两种方式：BINARY 和 COLLATE

  1. 在查询的字符串前面加上BINARY运算符
  2. 建表的时候对相应列加上BINARY
  3. 在查询时指定collation
  4. 在建表时指定collation

BINARY

BINARY 不是函数，而是一个类型转换运算符，用来强制将它后面的字符串修改为二进制字符串，这样在进行字符串的比较的时候就会区分大小写了

collate

在MySQL中，对于Column Collate其约定的命名方法如下：
| 命名方式 | 介绍 |
| - | - |
| _bin | 表示的是binary case sensitive collation，也就是说是区分大小写的 |
| _cs | case sensitive collation，区分大小写 |
| *_ci | case insensitive collation，不区分大小写 |
