算法设计方法

Posted on 2017-08-04 | In Note

总结一下常用的四种算法设计方法：分治、动态规划、贪心、回溯

分治法

分治法就是分而治之，把一个复杂的问题分成多个相同或相似的子问题，再把子问题分成更小的子问题…直到最后子问题可以简单的直接解决，原问题的解即子问题解得合并。

基本思想及策略

分治法的设计思想：讲一个难以直接解决的大问题，分割成一些规模较小的相同问题，各个击破，分而治之。

策略：对于一个规模为n的问题，如果无法直接解决，将其分解为k个规模较小的子问题，这些子问题互相独立并且与原问题形式相同，递归地解这些子问题，然后将各个子问题的解合并得到原问题的解。

分治法适用的情况

分治法能够解决的问题一般具有以下几个特征：

  1. 该问题的规模缩小到一定的程度就可以直接解决
  2. 该问题可以分解为若干个规模较小的相同子问题
  3. 分解出来的子问题的解可以合并成为该问题的解
  4. 各个子问题之间相互独立

第一个特征是分治法的前提，到一定程度有直接的解，作为递归的终点。
第二个特征表示可以通过递归来解决问题
第三个特征是分治法的关键，如果子问题的解不能合并成为原问题的解，就需要考虑使用其他的做法（贪心、动态规划）
第四个特征表示不会重复求解问题，如果有重复的子问题，用动态规划比较好

分治法的基本步骤

分治法在每一层递归上都有三个步骤：

  1. 分解：将原问题分解为若干个规模较小、相互独立、与原问题形式相同的子问题
  2. 解决：如果子问题可以直接解决就直接解决，否则递归地解子问题
  3. 合并：将各个子问题的解合并成原问题的解

分治法应用

快速排序
二分搜索
大整数相乘
Strassen矩阵乘法
汉诺塔

动态规划

动态规划所处理的问题是一个多阶段决策问题，一般由初始状态开始，通过对中间阶段决策的选择，达到结束状态。这些决策形成了一个决策序列，同时确定了完成整个过程的一条活动路线。

最优化原理

一个过程的最优决策具有这样的性质：无论其初始状态和初始决策如何，其今后诸策略对以第一个决策所形成的状态作为初始状态的过程而言，必须构成最优策略。

动态规划适用的情况

能采用动态规划求解的问题满足如下的条件：

  1. 问题中的状态必须满足最优化原理
  2. 问题中的状态必须满足无后效性

无后效性指的是：下一个状态只与当前状态有关，与当前状态之前的状态无关，当前的状态是对以往决策的总结。

动态规划基本步骤

  1. 划分阶段：根据问题的时间、空间特征，把问题划分成若干阶段。要保证划分后的阶段有序。
  2. 确定状态和状态变量：将问题发展到各个阶段时所处于的各种客观情况用不同的状态表示出来。状态的选择要满足无后效性。
  3. 确定决策并写出状态转移方程：状态转移就是根据上一个阶段的状态和决策来导出当前阶段的状态。事实上通常是反过来做，根据相邻两个阶段的状态之间的关系来确定这个转移方程。
  4. 寻找边界条件：给出的状态转移方程是一个递推公式，需要有一个终止条件。

动态规划的应用

0/1背包问题
最长公共子串问题

贪心算法

贪心算法是指在对问题求解时，总是做出在当前看来是最好的选择。也就是说，不从整体最优上加以考虑，所作出的仅仅是在某种意义上的局部最优解。

基本思想

  1. 把求解的问题分解成若干个子问题
  2. 对每个子问题求解，得到子问题的局部最优解
  3. 把子问题的局部最优解合并成一个解就是原问题的解

贪心算法不一定能够得到最优解，比如说对于背包问题，我们选取贪心策略为每次选取价值最大的，或者选取重量最小的，都不能保证拿到的整体价值最大。

贪心算法应用

最小生成树Prim算法
Prim算法从任意一个顶点开始，每次选择一个与当前顶点集最近的一个顶点，并将两顶点之间的边加入到树中。Prim算法在找当前最近顶点时使用到了贪婪算法。

  1. 将一个图的顶点分为两部分，一部分是最小生成树中的结点（A集合），另一部分是未处理的结点（B集合）。
  2. 首先选择一个结点，将这个结点加入A中，然后，对集合A中的顶点遍历，找出A中顶点关联的边权值最小的那个顶点（设为v），将此顶点从B中删除，加入集合A中。
  3. 递归重复步骤2，直到B集合中的结点为空，结束此过程。
  4. A集合中的结点就是由Prime算法得到的最小生成树的结点，依照步骤2的结点连接这些顶点，得到的就是这个图的最小生成树。
    Prim时间复杂度为O(N^2)，适合稠密图，因此通常使用邻接矩阵储存。

最小生成树Kruskal算法
Kruskal算法与Prim算法的不同之处在于，Kruskal在找最小生成树结点之前，需要对所有权重边做从小到大排序。将排序好的权重边依次加入到最小生成树中，如果加入时产生回路就跳过这条边，加入下一条边。当所有结点都加入到最小生成树中之后，就找出了最小生成树。

  1. 新建图G，G中拥有原图中相同的点，但没有边
  2. 将原图中所有的边按权值从小到大排序
  3. 从权值最小的边开始，如果这条边链接的两个点于图G中不在同一个连通分量中，则添加这条边到图G中
  4. 重复3，直至图G中所有的点都在同一个连通分量中

回溯法

回溯算法实际上一个类似枚举的搜索尝试过程，主要是在搜索尝试过程中寻找问题的解，当发现已不满足求解条件时，就“回溯”返回，尝试别的路径。

回溯法基本思想

在包含问题的所有解的解空间树中，按照深度优先搜索的策略，从根结点出发深度探索解空间树。当探索到某一结点时，要先判断该结点是否包含问题的解，如果包含，就从该结点出发继续探索下去，如果该结点不包含问题的解，则逐层向其祖先结点回溯。（其实回溯法就是对隐式图的深度优先搜索算法）。

若用回溯法求问题的所有解时，要回溯到根，且根结点的所有可行的子树都要已被搜索遍才结束；而若使用回溯法求任一个解时，只要搜索到问题的一个解就可以结束。

回溯法基本步骤

  1. 定义解空间，确定易于搜索的解空间结构
  2. 深度优先搜索解空间，在搜索过程中剪枝避免无效搜索

回溯法应用

八皇后问题

分支界限法

分支限界法常以广度优先或以最小耗费（最大效益）优先的方式搜索问题的解空间树。
类似于回溯法，也是一种在问题的解空间树T上搜索问题解的算法。但在一般情况下，分支限界法与回溯法的求解目标不同。回溯法的求解目标是找出T中满足约束条件的所有解，而分支限界法的求解目标则是找出满足约束条件的一个解，或是在满足约束条件的解中找出使某一目标函数值达到极大或极小的解，即在某种意义下的最优解。

分支搜索算法

所谓“分支”就是采用广度优先的策略，依次搜索E-结点的所有分支，也就是所有相邻结点，抛弃不满足约束条件的结点，其余结点加入活结点表。然后从表中选择一个结点作为下一个E-结点，继续搜索。
选择下一个E-结点的方式不同，则会有几种不同的分支搜索方式：FIFO搜索、LIFO搜索、优先队列式搜索
