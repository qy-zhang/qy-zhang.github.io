Effective Java 读书笔记

Posted on 2017-10-13 | In Note

Effective Java 读书笔记

创建和销毁对象

第1条：考虑用静态工厂方法代替构造器

类可以提供一个公有的静态工厂方法，它只是一个返回类的实例的静态方法。
静态工厂方法相比于构造器的优点：

  • 静态工厂方法有名称
    如果构造器的参数本身没有确切地描述正被返回的对象，那么具有适当名称的静态工厂会更容易使用，产生的客户端代码也更易于阅读。例如，构造器BigInterger(int, int, Random)返回的对象是一个整数对象，但是如果我们想要返回一个素数对象，使用静态工厂方法BigInterger.probablePrime来表示就更为清晰。
    我们也可以通过不同的构造器来实现，比如BigInterger(int, int, Random, bool isPrime);最后添加一个标志位判断是不是素数，这种做法虽然也能实现效果并且比较直观，但是API太多，对于用户来说太多的构造器会导致类的文档可读性变差。
  • 不用每次调用时创建一个新对象
    不可变类可以使用预先创建好的实例
  • 可以返回原返回类型的任何子类型的对象

遇到多个构造器参数时要考虑用构建器

当类的构造器有多个可选参数的时候，可以使用重叠构造器模式：第一个只有必要参数的构造器，第二个构造器有一个可选参数，第三个有两个可选参数，以此类推，最后一个构造器包含所有可选参数。这种做法客户端代码会很难编写，并且难以阅读，如果客户端不小心颠倒了其中两个参数的顺序，容易导致程序出现错误行为。

还有第二种替代方法，即JavaBeans模式：调用一个午餐构造器来创建对象，然后调用setter方法来设置每个必要的参数。这种做法创建实例很容易，但是客户端代码编写比较复杂，要显式调用多个函数。最严重的缺点是构造过程被分割到多个函数调用中，需要付出额外的努力来确保它的线程安全。

还有第三种做法，Builder模式：不直接生成想要的对象，而是让客户端利用所有必要的参数调用构造器，得到一个builder对象，然后客户端在builder对象上调用类似于setter的方法，来设置每个相关的可选参数，最后客户端调用午餐的build方法来生成不可变的对象。这个builder是它构建的类的静态成员类。

用私有构造器或者枚举类型强化Singleton属性

最基本的单例模式的写法，考虑到线程安全的问题，会用synchronized 关键字修饰getInstance()方法，另外还有饿汉式、懒汉式、静态内部类、双重校验锁的写法。

我们首先介绍两种实例单例的方法：这两种方法都要把构造器保持为私有的，并导出公有的静态成员，以便允许客户端能够访问该类的唯一实例。
第一种方法将静态实例设置成final，当类第一次加载到内存中的时候就初始化了，私有构造器仅被调用一次，用来实例化Elvis.INSTANCE这个静态成员。（类初始化顺序：父类->子类，静态->构造块->构造函数，类只有在被用到的时候才加载，类字节码加载后才可以被构造成对象实例）创建的实例是thread-safe。

注：享有特权的客户端可以借助AccessibleObject.setAccessible方法，通过反射机制调用私有构造器。如果需要抵御这种攻击，可以修改构造器，让它在被要求创建第二个实例的时候抛出异常。

第二种方法是静态工厂方法：通过getInstance方法调用，都会返回同一个对象的引用

第三种方法：只需要编写一个包含单个元素的枚举类型，单元素的枚举类型已经成为实现Singleton的最佳方法。

获取资源的方式通过调用Something.INSTANCE.getInstance()即可。
我们使用了enum来保证了构造方法为私有，在我们访问枚举实例时会自动执行构造方法，同时每个枚举实例都是static final类型的，也就标明只能被实例化一次。所以可以看出来，实际上通过枚举类型，表达的意思和第二种方法类似。
因为创建枚举默认就是线程安全的，你不需要担心double checked locking。

对于所有对象都通用的方法

尽管Object是一个具体类，但是设计它主要是为了扩展。它所有的非final方法（equals、hashCode、toString、clone和finalize）都有明确的通用约定，因为它们都被设计成是要被覆盖的。任何一个类在覆盖这些方法的时候，都有责任遵守这些通用约定；如果不能做到则会一点，其他依赖于这些约定的类（例如HashMap和HashSet）就无法结合该类一起正常工作。

覆盖equals时请遵守通用约定

如果不覆盖equals方法，类的每个实例都只与它自身相等。
equals方法实现了等价关系：

  • 自反性：对于任何非null的引用值x，x.equals(x)必须返回true
  • 对称性：对于任何非null的引用值x和y，当且仅当y.equals(x)返回true时，x.equals(y)必须返回true
  • 传递性：对于任何非null的引用值x、y和z，x.equals(y)同时y.equals(z)，那么x.equals(z)
  • 一致性：对于任何非null的引用值x和y，只要equals的比较操作在对象中所用的信息没有被修改，返回结果一直
  • 非空性：对于任何非null的引用值x，x.equals(null)必须返回false

使类和成员的可访问性最小化

设计良好的模块会隐藏所有的细节实现，把它的API与它的实现清晰地隔离开来。然后模块之间只通过它们的API进行通信，一个模块不需要知道其他模块的内部工作情况。这个概念被称为封装。
