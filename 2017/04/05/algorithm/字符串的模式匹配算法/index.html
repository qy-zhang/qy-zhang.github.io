字符串的模式匹配算法

发表于 2017-04-05 | 分类于 Note

子串的定位操作通常称作串的模式匹配。
在文本编辑中，我们经常要在一段文本中某个特定的位置找出 某个特定的字符或模式：
例：找出所有在文本T=abcabaabcaabac中的模式P=abaa所有出现。

kmp算法完成的任务是：给定两个字符串O和f，长度分别为n和m，判断f是否在O中出现，如果出现则返回出现的位置。常规方法是遍历a的每一个位置，然后从该位置开始和b进行匹配，但是这种方法的复杂度是O(n*m)。
kmp算法通过一个O(m)的预处理，使匹配的复杂度降为O(n+m)。

简单的字符串匹配算法

基本思想：从主串S的第pos个字符起和模式的第一个字符比较，若相等，则继续逐个比较后续字符；否则从主串的下一个字符起再重新和模式的字符比较。

实现：

使用暴力破解法，在最坏的情况下时间复杂度为O(nm)。我们考虑主串指针i的移动，每次都需要回溯，所以时间复杂度为乘的关系。
每次匹配失败都是从下一个位置开始匹配，所以遍历整个主串需要O(n)，每个位置都要从头到尾匹配模式串，所以每个位置都需要O(m)，整体最坏为O(nm)

我们主要把时间浪费在了回退的时间上。
如果我们事先知道模式串本身的信息就不用每次匹配失败后都把j回退回第一个去，
这种回退就浪费了很多不必要的时间，如果能事先计算出模式串本身的这些性质，
那么就可以在失配时直接把pattern移动到下一个可能的位置，把其中根本不可能匹配的过程省略掉。

KMP算法

上述的常规方法中，我们主要把时间浪费在了回退的时间上，每次都退回了起点，如果我们提前计算某些信息，就有可能一次前移多位。

通俗的来理解，就是说如果现在匹配到了第7个元素失配了，我们看前6个字符的最大长度相同前缀后缀。
假设字符串尾abcdab，我们发现最长的相同前缀后缀是ab，就直接把第三个字符’c’跟第7个元素比较。

所以重点就是如何得到这个最大长度的相同前缀后缀，也就是最关键的next数组。

next数组

我们希望得到最大长度的相同前缀后缀，这个数值比较好计算。
假设字符串为ABCDABD，对于每一位最大长度相同前缀后缀

字符 A B C D A B
长度 0 0 0 0 1 2

通过分析，我们知道当这串字符后面失配的时候，直接将开头的AB挪到最后的AB位置上，让字符C进行比较。总长度为6，最后一个B这个位置最大长度相同前缀后缀为2，我们需要把开头的A挪到最后面的A的位置上，需要挪动4个位置 = 6 - 2，那么可以总结出：
失配时，模式串向后移动的位数为：已匹配字符数 - 失配字符的上一位字符对应的最大长度值

而实际使用的时候，当前位置 = 已匹配字符数 + 1。
我们知道其实是在ABCDAB这六个字符后面的第七个字符来使用B这个位置的2，所以需要整体右移一位，最左边填-1，就有了next数组

字符 A B C D A B D
长度 0 0 0 0 1 2 0
next -1 0 0 0 0 1 2

所以总结可以修改一下，都使用当前位置，所以正好使用到了next数组
失配时，模式串向右移动的位数为：失配字符所在位置 - 失配字符对应的next值

那么如何通过程序来求解next数组呢？
我们假设已知next[0, … , j]，现在需要求解next[j+1]，已知有next[j] = k，有上面分析的右移知道next[j]表示的是前面j-1个字符的最大长度相同前缀后缀。
如果p[j] == p[k]，说明前面j个字符的最大长度相同前缀后缀可以再加1，所以next[j+1] = next[j]+1 = k+1
如果p[j] != p[k]，说明当前位置的最大长度相同的前缀后缀为0，如果我们这个时候直接将next[j+1]置为-1表示跳到开头，会没利用到next前面的信息，我们不用先急着跳到开头去。我们现在比较的是p[j]和p[next[j]]，可以递归看下去，p[next[next[j]]，直到到达了-1开头。

我们拿前缀 p0 pk-1 pk 去跟后缀 pj-k pj-1 pj 匹配，如果pk跟pj失配，下一步就是用p[next[k]]去跟pj继续匹配，如果p[next[k]]跟pj还是不匹配，则需要寻找长度更短的相同前缀后缀，即下一步用p[next[next[k]]]去跟pj匹配。此过程相当于模式串的自我匹配，所以不断的递归k = next[k]，直到要么找到长度更短的相同前缀后缀，要么没有长度更短的相同前缀后缀。如下图所示：

图中前k块和后k块相等，然后当p[j] != p[next[j]]时，即最右边的黄色不等于红色，我们想找最近的下一个匹配位置，图中四个蓝色都是相等的，在前k块里面在找到与最后一个蓝色相等的，所以就是比较p[j]和p[next[next[j]]]

KMP搜索

有了覆盖函数，那么实现KMP算法就十分简单，首先我们计算出next函数，然后将匹配失败的时候j的更新值更改即可。

KMP算法流程：
假设现在文本串s匹配到i位置，模式串t匹配到j位置。
如果j = -1，或者当前字符匹配成功s[i] == t[j]，就继续匹配下一个字符。
如果j != -1，并且当前字符匹配失败，则不动i，仅仅让j变为next[j]即可

next数组的优化

前面定义的next函数在某些情况下尚有缺陷。例如主串为”aaabaaaab”，模式串为”aaaab”，进行匹配时，在i=3、j=3时，发现s[3]=’b’，t[3]=’a’。
对于模式串，我们分析得到next数组：

字符 a a a a b
长度 0 1 2 3 0
next -1 0 1 2 3

所以根据next表分析，当在第四个位置不匹配的时候，此时next[3] = 2，就是说要向右滑动一位，但是我们知道前面三个都是字符’a’，是一样的，肯定还是不匹配的，所以我们希望能够跳到不是’a’的地方。
所以解决方式是只要t[i] == t[j]，就再next一遍

KMP算法的时间复杂度

通过KMP的过程可以看出，主串指针i只是一直在++，没有回溯，时间复杂度为O(n)，加上计算next数组的耗时O(m)，所以整体时间复杂度为O(n+M)

BM算法

KMP算法是从模式串的开头开始匹配，而BM算法从模式串的尾部开始匹配
BM算法拥有在最坏情况下O(N)的时间复杂度，在实践中比KMP算法的实际效能高。

BM算法定义了两个规则：坏字符规则、好后缀规则。
坏字符规则：当文本串中的某个字符跟模式串的某个字符不匹配时，我们称文本串中的这个失配字符为坏字符。此时模式串需要向右移动，移动的位数 = 坏字符在模式串中的位置 - 坏字符在模式串中最右出现的位置。此外，如果”坏字符”不包含在模式串之中，则最右出现位置为-1。

好后缀规则：当字符失配时，后移位数 = 好后缀在模式串中的位置 - 好后缀在模式串上一次出现的位置，且如果好后缀在模式串中没有再次出现，则为-1。

以一个例子来介绍BM算法流程：
主串”HERE IS A SIMPLE EXAMPLE”，模式串”EXAMPLE”

  1. 首先，”文本串”与”模式串”头部对齐，从尾部开始比较。”S”与”E”不匹配。这时，”S”就被称为”坏字符”（bad character），即不匹配的字符，它对应着模式串的第6位。且”S”不包含在模式串”EXAMPLE”之中（相当于最右出现位置是-1），这意味着可以把模式串后移6-(-1)=7位，从而直接移到”S”的后一位。
  2. 依然从尾部开始比较，发现”P”与”E”不匹配，所以”P”是”坏字符”。但是，”P”包含在模式串”EXAMPLE”之中。
  3. 因为“P”这个“坏字符”对应着模式串的第6位（从0开始编号），且在模式串中的最右出现位置为4，所以，将模式串后移6-4=2位，两个”P”对齐。
  4. 又从尾开始，发现从尾过来ELPM都对齐，但是I这里不匹配了，如果我们还是按照坏字符的方式，应该是将模式串后移2-(-1)=3位，但是明显M和E还是不匹配。有更有的做法：好后缀。
  5. MPLE、PLE、LE、E都是好后缀，后移位数 = 好后缀在模式串中的位置 - 好后缀在模式串中上一次出现的位置，只有E在模式串中上一次出现过了，出现在头部，所以后移位数 = 6 - 0 = 6
    可以看到，”坏字符规则”只能移3位，”好后缀规则”可以移6位。所以，Boyer-Moore算法的基本思想是，每次后移这两个规则之中的较大值。

Sunday算法

Sunday算法是从前往后匹配，在匹配失败时关注的是文本串中参加匹配的最末位字符的下一位字符。
如果该字符没有在模式串中出现则直接跳过，即移动位数 = 匹配串长度 + 1；
否则，其移动位数 = 模式串中最右端的该字符到末尾的距离+1。

下面举个例子说明下Sunday算法：
假定现在要在文本串”substring searching algorithm”中查找模式串”search”

  1. 刚开始时，把模式串与文本串左边对齐

  2. 结果发现在第2个字符处发现不匹配，不匹配时关注文本串中参加匹配的最末位字符的下一位字符，即字符i，因为模式串search中并不存在i，所以模式串直接跳过一大片，向右移动位数 = 匹配串长度 + 1 = 6 + 1 = 7，从 i 之后的那个字符（即字符n）开始下一步的匹配

  3. 结果第一个字符就不匹配，再看文本串中参加匹配的最末位字符的下一位字符，是’r’，它出现在模式串中的倒数第3位，于是把模式串向右移动3位（r 到模式串末尾的距离 + 1 = 2 + 1 =3），使两个’r’对齐，如下：
