字符串拷贝

Posted on 2017-04-17 | In Blog

字符串拷贝函数strcpy的实现也是面试中经常会考的问题。
函数声明为
char* strcpy(char* dst, const char* src);

strcpy实现

几个注意点：

const修饰源串

const修饰源字符串src，防止修改源字符串
同时支持const字符串作为参数传递，const转非const不允许，但是非const转const是允许

检查空指针

使用assert检查空指针，判断的时候最好写成assert(dst != NULL)
而不要写成assert(!dst)，或者是assert(dst!=0)
char*转换成bool是类型隐式转换，这种方式会导致出错概率增大和维护成本提高。
直接与常量0判断也不好，会减少程序的可维护性

在C语言中NULL和0的值都是一样的，在stdio.h头文件中包含了stddef.h，而在stddef.h中我们可以发现有如下代码

NULL通常被定义为0或(void*)0

而在C++中不支持void的隐式类型转换，所以NULL被明确定义为常数0，这样如果对于一个函数来说有两个重载函数`void fun(char)和void fun(int)， 现在如果调用的时候直接用NULL作为参数调用fun(NULL)`那么就不知道该调用哪个函数了。（测试中gcc会调用fun(int)函数）
在C++11中为了null指针不会隐式转换为整数，专门引入了nullptr

保存原始目标首地址并返回

返回dst的原始首地址是为了能够支持链式表达式
现在有如下需求，将strB复制到strA，然后求strA的长度，我们想一步到位
int l=strlen(strcpy(strA,strB));

循环的书写

循环的书写要考虑好结束条件
如果写成while (*src!='\0') *dst++=*src++;这样看起来没什么区别，实际上循环终止条件是到src尾就停止了，而没有拷贝结束符。
dst最后还要给他末尾加上’\0’

内存重叠的字符串拷贝

考虑dst和src内存重叠的情况，strcpy该怎么实现
所谓重叠，就是src未处理的部分已经被dst给覆盖了，只有一种情况：src<=dst<=src+strlen(src)
这个时候从后往前复制就好了
注意此时保存的dst的原始地址就是dst，而设置一个tmp地址从dst的尾走到头

实际上我们实现的my_memcpy是对应于C语言库函数string.h中的memmove函数，而实际上的memcpy函数时不考虑内存重叠
