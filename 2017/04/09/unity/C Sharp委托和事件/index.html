C Sharp委托和事件

委托

当我们要把方法传递给其他方法时，需要使用委托。

从需求上分析：
我们现在需要与别人打招呼，有两种方式，中文和英文的。

但是这样扩展性差，如果我们要新添加一种方式，必须修改GreetPeople类，比较麻烦。
我们肯定想将函数作为参数传入到GreetPeople函数中，这样就能够根据调用不同的函数，同时添加不同的函数都不用改变GreetPeople，只需要按照函数格式写好新函数即可。

但是将函数作为另一个函数的参数，这个参数的类型是什么呢？
在C++中，很明显，这是一个函数指针，而在C#中，这就是委托。
我们将GreetPeople改成如下所示：

使用C++的写法就是
public void GreetPeople(string name, (void*)MakeGreeting(string name));

在C++中，函数指针就是一个指针变量，但是在C#这种十分重视面向对象的语言中，委托是一个类，它定义了方法的类型。

多播委托

另外，在C#中，委托可以绑定多个方法，当调用绑定了多个方法的委托时，将依次调用其所绑定的方法。

但是注意，一定要先使用operator=对委托赋值，然后才能使用operator+=来对委托绑定多个方法。

Observer观察者模式

在设计模式中，Observer观察者模式是一对多的方式，可以让多个观察者同时关注同一个事务，并做出不同的响应。
开发 Observer 模式时借助委托，可以进一步简化开发的过程。注册、删除都变得更加方便。使用operator+=和operator-=即可。

事件

根据面向对象的思想，我们想尽量封装，我们想把委托封装起来，把GreetingDelegrate委托和GreetPeople封装到一个类中。

这样比较符合我们的要求了，但是还剩最后一个问题，封装性不够良好，要是类中的委托是private的就比较好了。但是我们本来就是要对委托赋值，来注册方法，现在又要将它声明成private，这样客户根本注册不了方法了。

这个时候，我们要使用事件了。

事件（event）可被视作为一种特别的委托，它为委托对象隐式地建立起add_XXX、remove_XXX 两个方法，用作注册与注销事件的处理方法。而且事件对应的变量成员将会被视为 private 变量，外界无法超越事件所在对象直接访问它们，这使事件具备良好的封装性，而且免除了add_XXX、remove_XXX等繁琐的代码。

事件能通过operator+=和operator-=两个方式注册或者注销对其处理的方法，使用operator+=与operator-=的时候，系统会自动调用对应的 add_XXX、remove_XXX 进行处理。

这也解决了上面提到的必须先对委托使用operator=赋值之后才能对委托使用operator+=多绑定。现在事件event直接规定在类的外部只能使用operator+=和operator-=来注册和删除。

只允许一个客户注册

如果我们想只允许一个客户注册，我们可以将事件声明称private的，这样外部不能访问，然后编写注册和删除函数，在类的外部对于事件event只能使用operator+=和operator-=，现在在类的内部，我们可以使用operator=了，我们在注册函数中使用operator=来保证只有一个客户

委托和事件的区别

事件就是一个狭义的委托。
你可以在客户代码中直接调用委托来使用委托指向的函数，而不能使用事件完成这样的操作，事件的触发只能由服务代码自己触发。
也就是说在你的代码里委托你不但可以安排谁是它的调用函数，还可以直接调用它；而事件不能直接调用，只能通过某些操作触发。
