C++ Rule of Three和RAII

发表于 2017-04-27 | 分类于 Blog

网上看到别人提到编写C++要注意几个规则：Rule of Three和RAII，总结一下就是说几个类的默认函数显式写出，初始化就是资源获取

Rule of Three

The rule of three and rule of five are rules of thumb in C++ for the building of exception-safe code and for formalizing rules on resource management. It accomplishes this by prescribing how the default members of a class should be used to accomplish this task in a systematic manner.


Rule of Three是指类如果需要需要拷贝构造函数、赋值操作符、析构函数中的任一个被显式定义，那么最好为这三个都显式定义。
在实际工程中最好为类显式的编写这三个函数，而不是使用系统默认定义的。这样操作可以有效的防止内存泄漏问题，对整个系统的结构有良好的把握。

用一个自定义的string类来作为例子。

拷贝构造函数

拷贝构造函数要注意是使用浅拷贝还是深拷贝。
浅拷贝要注意如果类对象拥有资源的情况下，释放对象可能造成的资源重复释放问题。
深拷贝往往是耗时耗内存的，考虑能否使用移动构造函数，直接与右值对象交换资源（作浅拷贝）

拷贝构造函数参数const引用，没有返回值（构造函数不允许有返回值）
如果不传引用，直接传递实参，那么在调用函数的时候要拷贝MyString对象作为副本传递进来，所以要调用拷贝构造函数，而本身就是在调用拷贝构造函数，所以会出现递归死循环调用。
如果不加const，那么就不能拷贝const对象

类在数组和容器中的使用要注意

对于第一个容器，先会调用MyString的含参构造函数，生成一个临时对象，然后调用拷贝构造函数将临时对象拷贝到vector中。

对于第二个数组，也先会调用MyString的含参构造函数，不过直接根据这个临时对象初始化数组的第一个数值。

RAII

RAII表示”Resource acquisition is initialization”，资源获取就是初始化。
实际上RAII是一种思想，实现资源自动管理，例如智能指针，将一个在堆上动态分配的对象，或者申请的资源，像在栈上的数据一样自动释放。
RAII的好处在于它提供了一种资源自动管理的方式，当产生异常、回滚等现象时，RAII可以正确地释放掉资源。
