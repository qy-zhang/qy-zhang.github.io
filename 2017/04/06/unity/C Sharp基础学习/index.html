介绍.net、CLI、CLR、IL、JIT

.net

.net是微软推出的框架，可以被看作是介于操作系统和编程语言之间的软件层，它可以支持多种编程语言，包括C#、VB.NET、C++、F#等等。
.net用来产生托管代码的框架、平台，传统的编译方式是源代码经过编译得到可执行文件，在.net中编译方式是源代码经过编译到程序集，在运行时再转换成为机器代码。

IL

IL(Intermediage Language)中间语言，通过.net框架编译出来的中间语言，还需要经过CLR进行编译才能转换成汇编语言来给CPU直接执行。

对应于Java来说就是字节码

CLR

CLR是公共语言运行环境(Common Language Runtime)，将IL中间语言编译成CPU能够直接执行的汇编代码。

CLR的工作流程内容：
管理应用程序域、加载和运行程序集、安全检查、将IL代码和JIT编译成机器代码、异常处理、垃圾回收等

对应于Java来说就是JVM

JIT

JIT(just in time)，即时编译，在CLR中将IL转换为机器代码的工作交给了JIT编译器，即时编译只在方法被第一次调用时发生。

程序内存分配

Windows使用虚拟寻址系统，把程序可用的内存地址映射到硬件内存中的实际地址，其作用是32位处理器上的每个进程都可以使用4GB的内存。这4GB内存称为虚拟内存。

C#的内存格局通常分为四个区：全局数据区、代码区、栈、堆。

堆栈和托管堆

  • 堆栈stack
    stack与C++中的栈相同，存放函数参数、局部变量，由编译器自动分配释放。
    存储值类型
    由高地址向低地址填充
    性能高

  • 托管堆heap
    heap与C++中的堆略有不同，C#中的托管堆在垃圾收集器的控制下工作，与传统的堆相比有很显著的性能优势。
    存储引用类型
    由低地址向高地址填充
    性能低

装箱和拆箱

在C#中将一个值类型转换为引用类型称为装箱；将引用类型转换为值类型称为拆箱。

装箱的过程：首先在堆栈中分配一个4个字节的空间来存储引用变量o，然后再托管堆中分配一定的空间来存储i的拷贝，并返回该内存地址。最后把这个地址赋值给变量o，o就是指向对象的引用了。o的值不论怎么变化，i的值也不会变，相反i的值变化，o也不会变，因为它们存储在不同的地方。

拆箱的过程：在堆栈上分配4个字节的空间来存储变量j，拷贝对象o实例的值到j的内存，赋值给j。

GC垃圾回收

引用类型的赋值相当于修改了指向的内存地址

这里我们会发现原始的a2对象没有引用指向他了，但是我们也没有对其进行回收。在C#中垃圾收集器会对这种数据进行处理。

垃圾收集器的基本算法原理如下：

  • 将所有的托管内存标记为垃圾
  • 寻找正被使用的内存块，并将他们标记为有效
  • 释放所有没有被使用的内存块
  • 整理堆以减少碎片

为了提升性能，遍历整个托管堆开销太大，为此加上了分代策略：
新分配的对象被放在generation0中，当generation0的大小快要达到它的上限的时候，一个只在generation0中执行的gc回收操作被触发。
由于generation0的大小很小，因此这将是一个非常快的GC过程。这个GC过程的结果是将generation0彻底的刷新了一遍。不再使用的对象被释放，确实正被使用的对象被整理并移入generation1中。
当generation1的大小随着从generation0中移入的对象数量的增加而接近它的上限的时候，一个回收动作被触发来在generation0和generation1中执行GC过程。如同在generation0中一样，不再使用的对象被释放，正在被使用的对象被整理并移入下一个generation中。

泛型的作用

通过C++来理解：泛型的语言支持机制就是模板templates，模板的思想就是参数化类型
把一个特定于某个类型的算法或类中的类型信息抽掉，抽出来做成模板参数T。有如下有点：

  • 通用性强，可以支持不同的类型，调用同一个函数、类
  • 效率高，在C++中模板是在编译期确定的，所以运行效率高

语法细节

internal

在访问控制方面，C#有public、protected、private、internal，其中internal是C++和Java所没有的。internal表示同一个编译集合（exe、dll）下的类可以相互访问。

实际中使用如下：
如果类的一个成员需要能被子类访问到，同时能被同一个集合（在Java中表示同一个包，C#中表示同一个编译集合）下的其他类访问到，在Java中直接使用protected即可；在C#中需要使用internal protected。
如果类的一个成员需要被子类访问到，但是不能被同一个集合下的其他类访问到，这个时候Java就无能为力了。C#中使用protected即可。

struct和class

在C++中可以使用struct和class来定义类型，两种方式基本一致，有微小区别。如果没有标明成员函数或者成员变量的访问权限级别，在struct中默认是public的，而在class中默认是private。

在C#中，如果没有标明成员函数或者成员变量的访问级别，struct和class中都是private的。struct和class的区别是struct定义的是值类型，而class定义的是引用类型。

构造函数

C#中派生类可以通过base来显式的调用基类的构造函数。

C#中可以为类型定义一个特殊的构造函数：静态构造函数。这个函数的特点是类型第一次被使用之前有运行时自动调用，而且保证只调用一次。

在调用类B的代码之前先执行了B的静态构造函数。静态构造函数先初始化类的静态成员变量，再执行函数体内的语句，因此先打印a1再打印a3。接下来执行主函数main中的B b = new B();，调用B的普通构造函数。构造函数先初始化成员变量，再执行函数体内的语句，因此先后打印出a2、a4。

根据C#的静态构造函数的特性，我们当要使用单例模式的时候，可以直接将类的构造函数设置为private，然后添加静态构造函数
