程序开发中常见错误

发表于 2017-04-11 | 分类于 Blog

总结一下程序开发中常见错误，以及出现的原因，解决方案

内存溢出Out of Memory

内存溢出就是你要求分配的内存超出了系统能给你的，系统不能满足
需求，于是产生溢出。

栈溢出

栈一般默认为1-2m，一旦出现死循环或者是大量的递归调用，在不断的压栈过程中，造成栈容量超过1m而导致溢出。

注意事项：

  1. 不要定义太大的静态数组，当静态数组比较大时，在堆上建立
  2. 尽量少使用递归，递归深度太深会导致栈溢出。改成迭代(iteration)，或者最好能写成尾递归，不用保存上一个调用的栈，得到值直接返回。
  3. 可以通过设置来增加程序栈的大小。

内存泄漏

申请了内存没有归还。这种问题最常见。
以发生的方式来分类，内存泄漏可以分为4 类：

  1. 常发性内存泄漏。发生内存泄漏的代码会被多次执行到，每次被执行的时候都会导致一块内存泄漏。
  2. 偶发性内存泄漏。发生内存泄漏的代码只有在某些特定环境或操作过程下才会发生。常发性和偶发性是相对的。对于特定的环境，偶发性的也许就变成了常发性的。所以测试环境和测试方法对检测内存泄漏至关重要。
  3. 一次性内存泄漏。发生内存泄漏的代码只会被执行一次，或者由于算法上的缺陷，导致总会有一块仅且一块内存发生泄漏。比如，在类的构造函数中分配内存，在析构函数中却没有释放该内存，所以内存泄漏只会发生一次。
  4. 隐式内存泄漏。程序在运行过程中不停的分配内存，但是直到结束的时候才释放内存。严格的说这里并没有发生内存泄漏，因为最终程序释放了所有申请的内存。但是对于一个服务器程序，需要运行几天，几周甚至几个月，不及时释放内存也可能导致最终耗尽系统的所有内存。所以，我们称这类内存泄漏为隐式内存泄漏。

如何发现内存泄漏

在Linux平台可以使用ps命令，来监视内存的使用：ps aux，aux参数表示显示所有包含其他使用者的进程。可以观测堆栈信息，增长趋势。top命令相当于任务管理器，实时动态刷新。

还可以使用静态代码检测工具，一般是lint工具。

new、delete成对使用

保证每个分配的内存被正确的释放掉。
同时delete释放对象数组的时候需要使用方括号。

return前释放内存

在多音调检测算法开发中，使用极端数据测试的时候发现内存一直在增长，在正常数据下内存就比较稳定。正常程序内存会比较稳定，如果一直增长一般是发生内存泄漏了。
检查代码后发现malloc分配的内存都对应有free语句释放，一直定位不到问题。之后才发现算法中有对极端条件下的处理，会直接return返回，但是这之前分配的内存就忘了释放掉，所以造成了内存泄漏。

多维数组的释放

在乐谱跟踪指纹算法中，我们定义了二维数组作为时间频率点对，并为二维数组分配了内存。

在C语言里二维数组有三种方式存放：int* ptr; int ptr[M]; int (ptr)[M];
第一种就是我们timePair用的那种，在定义的过程中编译器不会对他进行内存分配，完全由我们分配释放，最后要先每一维释放timePair[i]，然后在释放timePair
第二种int ptr[M]，编译器为ptr分配了内存，我们需要分配ptr[i]的内存，所以最后也只用回收这个的内存，对每一维释放ptr[i]，但是ptr由系统释放

有时候我们分配了一个大小为10的内存，然后没有free，接着给他赋值为大小为5，那么到后面，我们释放的时候只释放了5，那么就会有剩余的内存泄漏了

缓冲区溢出

由于计算机上的存储介质的速度是不一样的，寄存器最高，主存次之，最后是磁盘、光盘之类最低。这三种存储进行数据交换时，由于速度相差成千上万倍，故我们拿出一部分的存储作为暂时的存储区，而不是直接交换。
缓冲区溢出是指当计算机向缓冲区内填充数据位数时超过了缓冲区本身的容量溢出的数据覆盖在合法数据上，理想的情况是程序检查数据长度并不允许输入超过缓冲区长度的字符，但是绝大多数程序都会假设数据长度总是与所分配的储存空间相匹配，这就为缓冲区溢出埋下隐患。
