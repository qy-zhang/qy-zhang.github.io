C++ cast 四种显式类型转换关键字

发表于 2017-02-08 | 分类于 Note

在C语言中使用强制类型转换(Type Cast)很简单，不管什么类型的转换，形式都如下：TYPE b = (TYPE)a;
C++提供了4种类型转换操作符来应对不同场合的应用：const_cast、static_cast、dynamic_cast、reinterpreter_cast
C++风格的强制转换其他的好处是，它们能更清晰的表明它们要干什么。程序员只要扫一眼这样的代码，就能立即知道一个强制转换的目的。

const_cast

const_cast 去掉类型的const或volatile属性

对于const变量，我们不能修改它的值，这是这个限定符最直接的表现。但是我们就是想违背它的限定希望修改其内容怎么办呢？

这种方法显然达不到目的：对modifier的修改并不会影响到constant，这暗示了一点 const_cast转换符不该用在对象数据上，因为这样的转换得到的两个变量/对象并没有相关性。
只有用指针或者引用，让变量指向同一个地址才是解决方案

这是因为constant定义初始化为常量值21，这个21保存到了常量表中，在编译的时候会把constant替换成21，而不会对他取值

static_cast

static_cast 内置基本数据类型之间的转换以及对象之间的转换

  1. 基类和子类之间转换
    其中子类指针转换成父类指针是安全的;但父类指针转换成子类指针是不安全的。(类中不能包含虚函数)
  2. 基本数据类型转换
    enum, struct, int, char, float等。static_cast不能进行无关类型(如非基类和子类)指针之间的转换。

static_cast将子类对象转换为父类对象之后，只能调用父类对象中的成员变量，相当于子类扩展的部分没有作用了，没关系不会出现错误
但是如果是将父类对象转换为子类对象之后，子类对象扩展的部分没有被赋值，很危险

static_cast不能做两个没有继承关系的对象之间的转换

dynamic_cast

dynamic_cast 动态类型转换

其他三中强制转换都是编译时完成的，dynamic_cast是在运行时处理的，依赖于RTTI信息，运行时要进行类型检查。
使用dynamic_cast，基类中一定要有虚函数
许多编译器都是通过vtable找到对象的RTTI信息的，这也就意味着，如果基类没有虚方法，也就无法判断一个基类指针变量所指对象的真实类型，这时候，dynamic_cast只能用来做安全的转换，例如从派生类指针转换成基类指针。而这种转换其实并不需要dynamic_cast参与。也就是说，dynamic_cast是根据RTTI记载的信息来判断类型转换是否合法的。

在类的转换时，在类层次间进行上行转换时，dynamic_cast和static_cast的效果是一样的。在进行下行转换时，dynamic_cast具有类型检查的功能，比static_cast更安全。向上转换即为指向子类对象的向下转换，即将父类指针转化子类指针。向下转换的成功与否还与将要转换的类型有关，即要转换的指针指向的对象的实际类型与转换以后的对象类型一定要相同，否则转换失败。
当类中没有虚函数，dynamic_cast报错。当转换不安全，转换之后的对象指针被置为NULL

reinterpreter_cast

reinterpret_cast 重新解释类型

reinterpret_cast运算符是用来处理无关类型之间的转换；它会产生一个新的值，这个值会有与原始参数（expressoin）有完全相同的比特位。

从上边对reinterpret_cast介绍，可以感觉出reinterpret_cast是个很强大的运算符，因为它可以无视种族隔离，随便搞。

与C风格的强制类型转换能力相同。

  1. 转换内置数据类型为其他数据类型
  2. 转换一个指针为其它类型的指针
  3. 转换内置的数据类型为指针，无须考虑类型安全或者常量的情形。不到万不得已绝对不用。
    操作结果只是简单的从一个指针到别的指针的值的二进制拷贝。

总结

去const、volatile属性用const_cast。
基本类型转换用static_cast。
多态类之间的类型转换用daynamic_cast。
不同类型的指针类型转换用reinterpreter_cast。
