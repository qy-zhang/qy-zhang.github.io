C++ dynamic_cast和RTTI

Posted on 2017-06-09 | In Note

在C++中类型转换有四个关键字，其中dynamic_cast提供了类型安全检查，专门用于具有继承关系的类之间转换的，尤其是向下类型转换，是安全的。在多态类型间进行转换更提倡采用dynamic_cast。
dynamic_cast通过运行时类型信息（RTTI）来保证转换时类型安全的，类对象中存储着RTTI类型相关信息，比如类的名字、基类等。

RTTI

运行时类型信息(Run Time Type Information RTTI)用于在运行时识别类对象的信息。C++里面只记录的类的名字和类的继承关系链。使得编译成二进制的代码，对象可以知道自己的名字（ASCII），以及在继承链中的位置。
在读《C++编程思想》和《类的内存布局》这两本书时，我发现都提到了类的虚表中第一项是RTTI，只不过是虚表指针指向的是RTTI的下一个位置，就是第一个虚函数了。所以RTTI存放在在虚表中的最上面。
这也可以回答为什么dynamic_cast对应的类一定要有虚函数：RTTI存放在虚表的第一项，如果没有虚函数类都不会有虚表，就没有RTTI了，dynamic_cast就会报错。

typeid

typeid的主要作用就是让用户知道当前的变量是什么类型的。

typeid返回的是type_info类

如果操作数不是类类型或者是没有虚函数的类，则获取其静态类型；如果操作数是定义了虚函数的类类型，则计算运行时类型。
typeid最常见的用途是比较两个表达式的类型，或者将表达式的类型与特定类型相比较。

dynamic_cast

dynamic_cast会首先验证转换是否有效，只有转换有效，操作符才进行实际的转换。如果dynamic_cast转换指针类型失败，则返回0；如果转换引用类型失败，则抛出一个bad_cast类型的异常。
