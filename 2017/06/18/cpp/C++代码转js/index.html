C++代码转js

发表于 2017-06-18 | 分类于 Note

最近实验室同学在尝试将C++代码转换成为js代码，使得原先用C++开发的图像处理程序能够在浏览器中运行。我挺想知道是怎么转的，毕竟一个是典型的静态语言、编译型语言；而另一个是典型的动态语言、解释型语言。
我想肯定不是纯人工搬运，首先工程量不小，转完之后性能还是个大问题，图像处理fps得高，正常情况下处理图像数据计算方面js这门语言性能肯定是比不上C++的。最后得知是使用Asm.js进行转换的，不仅解决了手工重新实现算法的大工作量，还完美解决了转换到js性能降低的问题。
Asm.js性能比js好，肯定是做了哪些方面的优化，这些值得研究一下。（从名字上看就感觉不普通，asm汇编级别，那性能肯定不低）

Asm.js

asm.js是一个中間語言，设计目的是使采用C等编程语言编写的计算机软件可运行为网络应用程序，同时性能特征明显优于标准JavaScript。

Asm.js来自于JavaScript应用的一个新领域：编译成JavaScript的C/C++应用。它是JavaScript应用的一个全新流派，由Mozilla的Emscripten项目催生而来。
Emscripten将C/C++代码传入LLVM，并将LLVM生成的字节码转换成JavaScript。
具体的来说，Asm.js是JavaScript的一个子集，为性能优化而生。

asm.js通常不直接编写，而是作为一种通过编译器生成的中间语言，该编译器获取C++或其他语言的源代码，然后输出asm.js。

性能优化

我们知道js的性能是无法跟C++这种高效语言相比的，但是Asm.js比较注意性能优化，一般情况下，对于复杂的应用Asm.js的性能仅仅比普通C++编译的慢两倍（可以和Java或者C#相媲美）

Asm.js为了优化性能，做了一下几点：

  • 所有外部数据在一个称为堆的对象中存储并被引用。堆在本质上是一个大数组（应当是一个在性能上高度优化的类型化数组）。所有的数据在这个数组中存储——有效的替代了全局变量，结构体，闭包和其他形式的数据存储。
  • 能处理被挑出的几种不同的数值类型，而没有提供其他的数据类型（包括字符串，布尔型和对象）。
  • 当访问和赋值变量时，结果被统一的强制转换成一种特定类型。例如f = e | 0;给变量f赋值e，但它也确保了结果的类型是一个整数（|0把值转换成整数，确保了这点）。

通过以上几点，可以看出来虽然js语言是一门动态语言，在程序运行时变量的类型是不确定的，但是Asm.js没有这个问题，他确保程序运行时变量类型已知（可以转换），让js实现了静态语言的概念。
同时，在内存操作上，将变量存放在一大块内存上，相当于在栈上操作（实际上是堆）。

这种超过普通JavaScript的性能增益主要是由于100％的类型一致性以及几乎没有垃圾回收（内存是手动管理的大型类型数组）。这个更简单的模型没有动态行为，没有内存分配或释放，只有一组简单、定义明确的整数和浮点操作，从而可实现更好的性能和优化潜力。

之前看过云风提过自己实现“在堆上的栈”，我们知道在栈上操作内存方便，不用管理，程序会自动分配销毁内存，但是堆上的内存需要我们自己手动分配释放。如果我们能够在堆上自动分配释放内存就好了。我们可以这样操作：在堆上分配一大块内存，然后将这块内存视为栈空间操作，最后用完之后释放整块内存。

实际上这也是很多其他的语法糖的设计初衷，比如说类对象，类对象的使用就好像是使用一个build-in type的变量，定义一下就行了，不用管内存分配销毁。这些操作都放到了构造析构函数中去实现了。
智能指针也一样，指针指向的内存需要手动分配释放，比较麻烦且容易出错。我们知道普通的变量或者类对象能够像栈上的build-in type变量一样，自动管理内存，我们就把指针封装成一个类，给他加上构造函数和析构函数，那它就能够自动管理内存了。
