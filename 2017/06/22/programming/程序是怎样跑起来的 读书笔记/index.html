程序是怎样跑起来的 读书笔记

对程序员来说CPU是什么

CPU内部结构解析

CPU的内部由寄存器、控制器、运算器和时钟四个部分构成。
寄存器用来暂存指令、数据等处理对象，可以看作内存的一种。
控制器负责把内存上的指令、数据等读入寄存器，并根据指令的执行结果来控制整个计算机。
运算器负责运算从内存读入寄存器的数据。
时钟负责发出CPU开始计时的时钟信号。

对于程序员来说，寄存器是关注的重点。寄存器主要划分为八类：
| 种类 | 功能 |
| - | - |
| 累加寄存器 | 存储执行运算的数据和运算后的数据 |
| 标志寄存器 | 存储运算处理后的CPU状态 |
| 程序计数器 | 存储下一条指令所在内存的地址 |
| 基址寄存器 | 存储数据内存的起始地址 |
| 变址寄存器 | 存储基址寄存器的相对地址 |
| 通用计数器 | 存储任意数据 |
| 指令寄存器 | 存储指令 |
| 栈寄存器 | 存储栈区域的起始地址 |

函数调用机制

函数调用使用call指令，将函数的入口地址设定到程序计数器之前，然后把调用后需要执行的指令地址存储到栈中。
函数处理完毕之后执行return命令，将保存在栈中的地址设定到程序计数器中。

数据使用二进制数表示的

补码

二进制数表示负数时，会把最高位作为符号位使用，-1的二进制数不是10000001，而是11111111
负数补码就是将二进制数的个位取反，然后加1

逻辑右移和算数右移

逻辑右移高位补0，算数右移就是除2
例如将-4（=11111100）逻辑右移两位，结果会变成63（=00111111），而算数右移结果应该是除两次2，变为-1

对于左移没有这个困扰，左移实际上都是补0。

内存和磁盘的亲密关系

磁盘中存储的程序，必须要加载到内存中才能运行。
磁盘缓存可以把磁盘中独处的数据存储到内存空间中，这样就可以改善磁盘数据访问速度慢的问题了。
可以使用虚拟内存将磁盘作为部分内存来使用。有分段和分页几种方式。

节约内存的编程方法

  1. 通过dll文件共享函数
    同一个dll文件的内容在运行时可以被多个应用公有，因此内存中存在函数的程序只有一个，不用多次拷贝函数，内存空间暂用小。同时可以在不变更exe情况下升级dll更新程序
  2. 通过调用_stdcall来减小程序文件的大小
    C语言调用函数后，需要执行栈清理处理指令，把不需要的数据从接收和传递函数的参数时所使用的内存从栈区域清理掉。在函数前加上_stdcall，可以把栈清理处理变为在被调函数一方进行
