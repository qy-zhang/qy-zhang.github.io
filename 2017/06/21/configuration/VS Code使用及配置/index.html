命令使用

ctrl + shift + p或F1调出命令面板，可以访问VS Code所有功能

ctrl + p打开窗口，可以干很多事情
直接输入文件名，快速打开文件

  • ? 列出当前可执行的动作
  • ! 显示Errors 或Warnings，也可以ctrl+shift+m
  • : 跳转到行数，也可以ctrl + g直接进入
  • @ 跳转到symbol（搜索变量或者函数），也可以ctrl + shift + o直接进入
  • @: 根据分类跳转symbol，查找属性或函数，也可以ctrl + shift + o后输入:进入

  • 根据名字查找symbol，也可以ctrl + t

开发环境配置

一些实用插件

code runner

安装插件code runner，能够一键运行各种语言的代码
安装完成之后配置code runner的编译选项，在 文件->首选项->设置 中编辑用户配置文件
比如说默认的c/c++编译选项是没有添加-g -Wall的，所以这里我们修改对应的位置，添加上，同时对于c++添加上-std=c++11来支持c++11特性。

以cpp为例分析编译命令：cd $dir && g++ -g -Wall -std=c++11 $fileName -o $fileNameWithoutExt && start cmd /c \”$fileNameWithoutExt & pause\”
首先code runner会在命令行下执行，上面的命令就是bat命令格式，&&表示接下来运行下一条命令

cd $dir 切换到当前文件所在目录下
g++ -g -Wall -std=c++11 $filename -o $fileNameWithoutExt 用g++编译（附加调试信息，支持C++11特性）当前文件，编译文件后生成同名的可执行文件
start cmd /c \”$fileNameWithoutExt & pause\” 新开一个命令行，/c后面是一串字符串参数，作为启动cmd的参数
重点是这一句话，我们希望在新开的命令行中运行程序，然后我们按任意键退出命令行，所以使用/c参数，表示执行完成之后推出命令行，但是如果直接退出会看不到输出结果，所以需要在命令行中暂停等按下任意键退出，需要& pause。执行程序和暂停要作为一整个参数输入给新开的命令行，所以要将他们拼接成一个字符串，\是转义符，用两个双引号将他们拼接起来就行了。

通过上述配置可以编译程序，然后在命令行中运行程序（可以通过控制台输入）

C++开发环境

VS Code自带调试器，能够方便的调试代码，同时有很良好的界面观察调试中的变量。
安装C++插件：C/C++、C/C++ Clang Command、C/C++ Snippets、C++ Intellisense

如果是在windows下，编译器使用mingw，推荐使用TDM-GCC，安装好之后配置环境目录。
点击右边的蜘蛛debug，再点击左边调试栏上方的设置按钮，选择c++编译环境，然后编辑launch.json文件

保存之后切换到cpp文件，按下F5进行调试，会弹出信息框要求配置任务运行程序，选择others编辑tasks.json配置文件
