SSH学习和使用

SSH介绍

SSH为Secure Shell的缩写，专为远程登录会话和其他网络服务提供安全性的协议。

传统的网络服务程序:ftp、pop、telnet在网络上用明文传送口令和数据。SSH可以把所有传输的数据进行加密压缩，能够安全又高速的完成传输功能。

SSH采用了公钥加密算法来保证传输安全。公钥加密流程如下

  1. 远程主机收到用户的登录请求，把自己的公钥发给用户。
  2. 用户使用这个公钥，将登录密码加密后，发送回来。
  3. 远程主机用自己的私钥，解密登录密码，如果密码正确，就同意用户登录。

公钥计算使用RSA算法，长度长达1024位

SSH使用

远程登录服务器使用ssh命令。以用户名user登录远程服务器host，使用命令
ssh user@host
ssh的默认端口为22，可以通过 -p 参数来设置端口号：ssh -p 5252 root@127.0.0.1

在登录到远程主机的时候，都会提示一段话

表示无法确认host主机的真是行，只知道他的公钥指纹，当输入yes之后会添加host主机到信任名单。
接着输入远程服务器用户对应的密码即可登录了。

当远程服务器的公钥被接受之后，这个公钥会被保存到文件中，存到 $HOME/.ssh/known_hosts 目录下。
当下次再连接这台主机的时候就会跳过上述的提醒直接提示输入密码登录了。

Windows下使用ssh

在windows下可以使用 putty + winscp 来通过ssh操作服务器。
winscp会读取putty中保存的密钥，能够十分方便的在本地和服务器之间进行文件操作。

特别推荐一款chrome扩展 Secure Shell
它在chrome标签页中模拟了一个终端，能够连接到ssh管理服务器。
能够同步保存ssh的配置（每次登录ssh需要对应用户的密码），支持命令行中复制粘贴。
与winscp相比起来不能够方便的上传下载文件了，但是能够通过scp命令来完成上传下载功能。

Mac下使用ssh

在mac的终端中可以直接使用ssh命令，系统中自带ssh
通过 whereis 命令可以查看ssh的位置

但是如果使用ssh命令来连接远程服务器或者连接本地主机的时候会报错
ssh: connect to host localhost port 22: Connection refused

mac系统中默认是关闭远程登录选项的，需要自己在设置中打开
在 系统偏好设置 -> 共享 -> 远程登录 中将远程登录属性框勾选即可

在mac中开启远程登录选项之后，搭配chrome扩展 Secure Shell，能够方便的进行远程登录操作。
上传下载文件需要使用scp命令，在mac终端中使用scp命令来完成服务器与本地之间的文件交互操作。

PS： scp命令介绍

SSH连接虚拟机Ubuntu

在windows下安装了virtualbox，虚拟出一台ubuntu虚拟机，但是只是想操作一下命令行就行了。如果直接在虚拟机中操作需要在windows和virtualbox两个程序之间来回切换，比较麻烦，我们可以直接通过ssh来登录本机创建的虚拟机。
参考博客
主要是添加一个端口转发规则，在virtualbox的 设备 - 网络 - 网络 中进行添加
网卡1连接方式为网络地址转换NAT，点击高级，显示高级设置
点击端口转发按钮打开端口转发规则列表
添加一条 名称为ssh，协议为TCP，本机端口为2222，子系统端口为22

然后在windows中ssh username@127.0.0.1:2222就可以登录了
