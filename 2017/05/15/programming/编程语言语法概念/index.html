编程语言语法概念

发表于 2017-05-15 | 分类于 Note

在面试中通常会问一下编程语法中的概念，在实际编程中也需要对这些概念有清晰的认识。

类型系统

强弱类型

强类型的语言遇到函数引数类型和实际调用类型不匹配的情况经常会直接出错或者编译失败；而弱类型的语言常常会实行隐式转换，或者产生难以意料的结果。

强类型：偏向于不容忍隐式类型转换。比如说haskell的int就不能变成double
弱类型：偏向于容忍隐式类型转换。比如说C语言的int可以变成double

Java、C#、Python都是强类型语言，C/C++、JavaScrpit、PHP等都是弱类型语言。

动态静态语言

静态类型：编译的时候就知道每一个变量的类型，因为类型错误而不能做的事情是语法错误。(compile time)
动态类型：编译的时候不知道每一个变量的类型，因为类型错误而不能做的事情是运行时错误。(run time)譬如说你不能对一个数字a写a[10]当数组用。

静态类型语言是在编译期间检查数据类型，所以在编写程序时要声明所有变量的类型，声明方式分为显式和隐式两种：显式声明就是直接在程序中显式写出类型的名字int a；隐式声明式在编译时类型推导。
动态类型语言是在运行期间检查数据类型，开始不需要指定数据类型，变量在第一次被赋值时解释器在内部将数据类型记录下来。

编译器解释器

从字面上看，“编译”和“解释”的确都有“翻译”的意思，它们的区别则在于翻译的时机安排不大一样。打个比方：假如你打算阅读一本外文书，而你不知道这门外语，那么你可以找一名翻译，给他足够的时间让他从头到尾把整本书翻译好，然后把书的母语版交给你阅读；或者，你也立刻让这名翻译辅助你阅读，让他一句一句给你翻译，如果你想往回看某个章节，他也得重新给你翻译。

多态

在面向对象方法中一般是这样表述多态性的：向不同的对象发送同一个消息， 不同的对象在接收时会产生不同的行为（即方法）

运行期多态

运行期多态的设计思想要归结到类继承体系的设计上去。
实现依赖于虚函数机制。当某个类声明了虚函数时，编译器将为该类对象安插一个虚函数表指针，并为该类设置一张唯一的虚函数表，虚函数表中存放的是该类虚函数地址。运行期间通过虚函数表指针与虚函数表去确定该类虚函数的真正实现。

运行期多态通过虚函数发生于运行期。属于晚绑定

编译期多态

编译期多态分为三种：函数重载、函数覆盖、模板。
函数重载：相同作用域内，函数名称相同，参数不同
函数覆盖：派生类拥有与基类同名同参数的函数，派生类中会屏蔽掉基类的函数

对模板参数而言，多态是通过模板具体化(specialization)和函数重载解析实现的。以不同的模板参数具体化导致调用不同的函数，这就是所谓的编译期多态。
相比较于运行期多态，实现编译期多态的类之间并不需要成为一个继承体系，它们之间可以没有什么关系，但约束是它们都有相同的隐式接口。

在编译之前，函数模板中t.shout()调用的是哪个接口并不确定。在编译期间，编译器推断出模板参数，因此确定调用的shout是哪个具体类型的接口。不同的推断结果调用不同的函数，这就是编译器多态。这类似于重载函数在编译器进行推导，以确定哪一个函数被调用。

内存错误

内存泄漏

申请了内存没有归还。
本质上来说，就是new了一个对象，但是没有用delete去删除它。

内存溢出

堆溢出：malloc内存过大，超过系统内存4G
栈溢出：stackoverflow，栈一般默认为1~2M，一旦出现死循环或者是大量的递归调用，在不断的压栈过程中，造成栈容量超过1M而导致溢出。
注意事项：

  1. 不要定义太大的静态数组，当静态数组比较大时，在堆上建立
  2. 尽量少使用递归，递归深度太深会导致栈溢出。改成迭代(iteration)，或者最好能写成尾递归，不用保存上一个调用的栈，得到值直接返回。
  3. 可以通过设置来增加程序栈的大小。

栈满了的时候push一个就会产生溢出，叫做上溢，栈空的时候pop，叫做下溢。
缓冲区溢出是一种特殊的内存溢出（广义），导致缓冲区溢出的原因就在于不检查数组容量和下标。

缓冲区溢出攻击是利用函数调用过程中的进出栈原理，在调用函数的时候，就把函数返回地址写入到一个栈上，然后新函数的局部变量写入到了这块内存所在的栈中，因为栈是自下而上增长的，所以不断写就会覆盖掉函数返回地址所在的那块内存，然后当函数返回的时候，就会执行到那块内存中指向的位置的代码，在该指向位置插入自己所想被执行的代码就可以完成缓冲区溢出攻击。
