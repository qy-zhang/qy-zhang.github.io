STL仿函数学习

发表于 2017-05-27 | 分类于 Note

仿函数(functor)又被称为函数对象(function object)，是指一种具有函数性质的类对象。换句话说就是让一个类的使用看上去像是一个函数。
其实现就是类中实现一个operator()，在C++中，函数调用操作符是指左右小括弧() ，该操作符是可以重载的（复习一下不能重载的操作符 . .* :: ?: sizeof typeid）。这个类就有了类似函数的行为，就是一个仿函数类了。

总结一句就是：通过重载()运算符模拟函数行为的类。

要注意两点：

  1. 仿函数不是一个函数，而是一个类
  2. 仿函数重载了operator ()运算符，使得调用形式好像是在调用函数

    仿函数在STL算法中的应用

    STL所提供的各种算法，往往有两个版本，其中一个版本表现出最常用（或最直观）的某种运算，第二个版本则表现出最泛化的演算流程，允许用户“以template参数来指定所要采取的策略”。

仿函数的优势

  • 仿函数是一个函数对象，能够拥有状态。
    函数对象可以将附加数据保存在成员变量中，从而实现携带附加数据，而函数指针就不行了。

比如：我们需要使用std::for_ecah将一个std::vector中的每一个值加上某个值然后输出，如果使用普通函数，则其声明应该为void add_num(int value, int num);，其中value为容器中的元素，num为要加上的数。但是由于std::for_each函数的第三个参数要求传入接受一个参数的函数或函数对象，所以将add_num函数传入std::for_each是错误的，然而函数对象可以携带附加数据解决这个问题。

  • 仿函数有自己的类型
    仿函数是类对象，有自己的类型，而函数指针只是一个指针。

  • 仿函数速度比普通函数更快
    对于模板概念来说，很多细节在编译期间就已经确定了，能够提升运行速度。

仿函数与lambda

如果用lambda实现上述代码，更为简单

lambda表达式会让编译器产生一个类似前面例子中AddClass的未命名函数对象类，并且具有以下优点：

  1. 简洁，不需要自己去额外实现一个仿函数
  2. 不会导致名字污染，没有实现仿函数，就没有引入新的名字
  3. 简单的代码放在靠近使用的地方能够提高代码清晰度

lambda语法参看博客，简单的来说就是
[capture](params) -> ret {body}
capture捕获外部变量（传值、传引用都可以），传入参数params，调用函数体body，ret返回结果（可以由return语句隐式决定）

在上面的for_each语句中，最后的lambda函数的capture部分是空[]，看起来好像没有捕获外部变量，那他是如何对vector中的每个值遍历处理的呢？实际上我们查看for_each的源码

std::function

lambda产生的闭包类型可以转换成std::function。std::function对象是对C++中现有的可调用实体的一种类型安全的包裹。
std::function是个类模板，可以对函数（普通函数、成员函数）、lambda表达式、std::bind的绑定表达式、函数对象等进行封装。std::function的实例可以对这些封装的目标进行存储、复制和调用等操作。

来lambda表达式产生的临时对象叫做闭包对象。类型的匿名性并不影响std::for_each的调用，因为它是一个函数模板，会进行类型推导。

std::bind

函数模板std::bind能对普通函数、成员函数、静态成员函数、公共成员变量、公共静态成员变量等进行包装，调用std::bind的包装相当与将函数名和参数绑定在函数内部。std::bind函数模板返回的函数对象的类型是不确定的，但是可以存储在std::function内。std::bind绑定的参数是通过传值的方式传递的，如果需要通过引用传递则参数先需要用std::ref、std::cref进行引用，然后在传递给std::bind 将std::bind函数模板的返回值保存在std::function后，调用时需要传递的参数个数由std::bind中的占位符（std::placeholders::1、std::placeholders::2、std::placeholders::_3等）个数决定，即有几个占位符调用时就需要几个参数。
