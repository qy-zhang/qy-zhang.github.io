素数判断

Posted on 2017-05-23 | In Note

素数判断是一个很常见的算法，在实际中也应用广泛。
素数又称质数：一个大于1的自然数，除了1和它本身外，不能被其他自然数（质数）整除

试除法

根据定义我们可以很容易分析出，从2到n-1遍历每个整数去除，如果能够整除，不是素数；如果均无法整除，则是素数。

我们稍加分析很容易知道，从根号n到n-1这些数都不可能被n整除，所以遍历终止条件不用设置为n-1，可以设置为sqrt(n)。
我们知道偶数肯定不是素数，所以可以把步长设置为2，不用每个数字都去遍历了
开始值从3判断，所以前面加上对2的处理

尝试从3到sqrt(x)的所有奇数，还是有些浪费。比如要判断101是否质数，101的根号取整后是10，那么需要尝试的奇数分别是：3，5，7，9。但是实际上对9的尝试是多余的。不能被3整除，必然不能被9整除……其实，只要尝试小于sqrt(x)的质数即可。而这些质数，恰好前面已经算出来了。我们花费一些空间时间复杂度，将前面计算出来的素数存下来，用于后面的试除遍历。

筛选法

比如从1到100，逐个判断是否是素数。
先将1挖掉（1不是素数）
用2去除他后面的各个数，把能被2整除的数挖掉，倍数都不是素数
3的倍数挖掉
……
持续到n-1

确定质数范围

如果是求1到一百万之间的质数，那么筛选法的范围上线就是一百万即可，但是如果是求前100个质数，那么就得分析所有质数的分布范围了。
质数的分布是越往后越稀疏，有一个质数定理：

一个个地看，素数在正整数中的出现没有什么规律。可是总体地看，素数的个数竟然有规可循。1到x中的质数大约有f(x) = x/lnx个

根据质数定理可以知道x/lnx = 100，就能解出分布范围了。

设计存储容器

需要设计一个存储容器，来存储该分布范围内的所有自然数，然后在筛选的过程中将不符合要求的数值删除。
我们直观的想法是使用一个动态数组来存放每一个数字，不符合要求将其删除。这样做明显又耗内存，又不断的在拷贝内存，效率极低。
所以可以使用bool数组，bool的索引表示每个自然数，如果不符合要求就将对应的bool位置为false
实际上还可以进一步节省空间，使用bitmap，每一个bit位表示一个自然数，不符合要求将其置为二进制0
