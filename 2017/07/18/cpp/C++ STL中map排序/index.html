C++ STL中map排序

发表于 2017-07-18 | 分类于 Note

map是用来存放(key, value)键值对的数据结构，可以很方便快速的根据key查到相应的value。
假如存储学生姓名（假设不会重复）和其成绩，就可以定义map<string, int>来存储，map会自动根据map->first进行排序，所以最后如果使用iterator来遍历输出会是按照string排序输出。

如果我们想要按照名字顺序倒序输出呢？或者是按照成绩map->second排序输出呢？

map底层实现

我们知道在STL中map底层用的是红黑树，只是在红黑树上加了一层封装。

从map模板类定义中可以看到四个参数，第一个参数和第二个参数分别就是我们的key和value的类型。
第三个参数Compare类对象，实际上他是一个仿函数（函数对象），重载了operator()，在这里是作为了对Key的排序方式，有一个默认值less，说明map默认是升序排序的。
第四个参数是Allocator类，指明map空间配置器，默认不需要修改。

排序方式

map的第三个参数是一个函数对象，指明了对map中的Key的排序方式，默认是一个简单的less仿函数

按Value排序

通过对Compare仿函数的自定义，我们可以自定义比较函数来对Key进行排序，来实现我们想要的效果，但是如果我们想要对Value进行排序，修改Compare仿函数就无法实现了。

stl中sort函数只能对序列式容器进行排序，同时还要要求能够随机访问容器，所以对于list不能使用sort函数（list自己使用归并排序实现了sort函数list.sort()），同时红黑树这种非线性存储的容器也不能使用sort进行排序。

我们可以将map中的pair放到vector中去，然后调用sort函数排完序之后，输出整个vector即可。
要对序列容器中的元素进行排序，也有个必要条件：就是容器中的元素必须是可比较的，也就是实现了<操作的。

重点关注operator<的实现
第一种情况：x.first < y.first 这个好理解，就是比较key，如果x的key 小于 y的key 则返回true
第二种情况有点费解：!(y.first < x.first) && x.second < y.second，就是说首先不满足第一种情况，说明x.first >= y.first，现在又要!(y.first < x.first)，说明y.first == x.first，然后比较两者的second，说明是当x和y的key相等的时候，就去比较两者的value
为什么不直接写 x.first == y.first 呢？
作为map的key必须实现<操作符的重载，但是并不保证==符也被重载了，如果key没有提供==，那么 ，x.first == y.first 这样写就错了。

现在我们知道了pair类重载了<符，但是它并不是按照value进行比较的，而是先对key进行比较，key相等时候才对value进行比较。
如果想要实现我们的对value进行比较，就要重载这个operator<，但是pair已经实现了这个了，再去重载会出错。
