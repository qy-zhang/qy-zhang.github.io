数据结构-查找

发表于 2017-03-20 | 分类于 Note

查找算法和数据结构

查找表

查找表是由同一类型的数据元素（或记录）构成的集合。
对查找表经常进行的操作有：

  1. 查询某个“特定”的数据元素是否在查找表中
  2. 检索某个“特定”的数据元素的各种属性
  3. 在查找表中插入一个数据元素
  4. 在查找表中删除某个数据元素

若对查找表只作前两种统称为“查找”的操作，则称此类查找表为“静态查找表”。若在查找过程中同时插入查找表中不存在的数据元素，或者从查找表中删除已存在的某个数据元素，则称此类表为“动态查找表”。

查找操作的性能分析

衡量一个算法好坏的度量有3条：时间复杂度、空间复杂度、算法的其他性能。

定义：为确定记录在查找表中的位置，需和给定值进行比较的关键字个数的期望值成为查找算法在查找成功时的平均查找长度(Average Search Length)。

静态查找

顺序查找

从表中第一个记录开始，逐个进行记录的关键字与给定值比较
长度为n的表的平均查找长度为 ASL = 1/n * (1+2+3+…+n) = (n+1)/2

折半查找

要求表中元素必须是有序的，如果是无序的须先进行排序操作

用给定值k先与中间结点的关键字比较，中间结点把线形表分成两个子表，若相等则查找成功；若不相等，再根据k与该中间结点关键字的比较结果确定下一步查找哪个子表，这样递归进行，直到查找到或查找结束发现表中没有这样的结点。

不进行插入删除操作，属于静态查找表
复杂度分析：最坏情况下，关键词比较次数为log(n+1)

次优查找树

上述的两种查找都是在每个记录等概率下进行，如果每个记录的查找概率并不相等，那么查找成功的查找性能是其带权内路径长度之和。
PHi = wi * hi wi表示权值，hi表示所在层数，对PHi求和
使得PH值最小的二叉树为静态最优查找树(Static Optimal Search Tree)。由于构造静态最优查找树花费的时间代价较高，我们现在构造次优查找树，使得这棵二叉树的带权内路径长度PH值在所有具备同样权值的二叉树中近似为最小。
构造方法：

每次根结点的选取条件是左右子树的权重和差值最小，即左右都比较平衡。最后还需微调，防止被选为根结点的关键字比它相邻的关键字的权值小，导致大权值在更深的层次，使得wi*hi变大。微调要保持中序遍历结果不变。

动态查找

动态查找特点：表结构本身是在查找过程中动态生成的。即对于给定值key，如果表中存在其关键字等于key的记录，则查找成功返回；否则插入关键字等于key的记录。

二叉排序树

二叉排序树或是一棵空树，或具有如下性质：树的左子树上所有结点的值小于根结点的值，右子树的所有结点值大于根结点的值，左右子树也是二叉排序树。

插入一个结点比较容易，删除一个结点需要调整，删除以后仍然要保持二叉树的特性。
删除分三种情况讨论：

  1. 删除结点为叶子结点，不会破坏树结构，直接清空释放内存即可。
  2. 删除结点只有左子树(右子树)，此时将其左子树(右子树)候补上来即可。也不会破坏树的结构。
  3. 删除结点左右子树都有，为了保持中序遍历其他元素相对顺序不变，此时有两种做法
    方法一：从左子树中选，选左子树中最右边的结点补上(如果最右边有左子树，将左子树补上那个结点)
    方法二：从右子树中选，选右子树中最左边的结点补上

复杂度分析：二叉查找树与折半查找一样，插入和查找的时间复杂度均为O(logn)，但是在最坏的情况下仍然会有O(n)的时间复杂度。我们追求的是在最坏的情况下仍然有较好的时间复杂度，这就是平衡查找树设计的初衷。

注：判断一棵二叉树是否是二叉排序树
对二叉排序树进行中序遍历，当前结点的值总是大于前驱结点的值。遍历一边二叉树，保存其前驱结点的值即可。

平衡二叉树

平衡二叉树(Balanced Binary Tree)又称AVL树。它或者是一颗空树，或者是具有下列性质的二叉树：它的左子树和右子树都是平衡二叉树，且左子树和右子树的深度之差的绝对值不超过1。若将二叉树上结点的平衡因子(Balance Factor)定义为该结点的左子树的深度减去他的右子树的深度，则平衡二叉树上所有结点的平衡因子只能是-1、0和1。

当平衡的二叉排序树因插入结点而失去平衡时，仅需对最小不平衡子树进行平衡旋转处理。
旋转可归纳为下面4中情况：

  1. LL型，某结点的左结点的左子树上插了一个新结点。
    此时将树向右旋一次即可。
  2. RR型，某结点的右结点的右子树上插了一个新结点。
    此时将树向左旋一次即可。
  3. LR型，某结点的左结点的右子树上插了一个新结点。
    此时需要旋转两次，先左旋再右旋
  4. RL型，某结点的右结点的左子树上插了一个新结点。
    此时需要旋转两次，先右旋再左旋

B-树

B-树是一种平衡的多路查找树，他在文件系统中很有用，主要用于文件的索引。
(n, A0, K1, A1, K2, A2, … , Kn, An)
其中：Ki为关键字，Ai为指向子树根结点的指针。

树种每个结点最多有m颗子树（m为B-树的深度），最少有[m/2]颗子树，如果插入超过了m就需要分裂，删除小于了[m/2]就需要合并。

在B-树上进行查找包含两种基本操作：在B-树中找结点（磁盘）、在结点中找关键字（内存）。总结来说就是先在磁盘上找到指针p所指的结点后，将结点中的信息装入内存，利用顺序查找或折半查找查询等于K的关键字。
在计算机中，在磁盘上进行一次查找比在内存中进行一次查找耗时多得多，因此在磁盘上查找的次数，即待查关键字所在结点在B-树上的层数是决定B-树查找效率的首要因素。

B-树可以进行插入删除操作，属于动态查找表。插入需分裂、删除需合并以保持性质。

关键字的个数n必须满足： [ceil(m / 2)-1]<= n <= m-1 （ceil为取上整）

B+树

与B-树的区别：

  1. 多一个指向关键字最小的叶子结点的指针，可以从最小关键字起顺序查找
  2. 查找成功也要从根走到叶子结点。

由于有了指向最小叶子结点的指针，B+树有两种查找运算：一种是从最小关键字起顺序查找；另一种是从根结点开始，进行随机查找。
B+树不管查找成功与否，每次查找都是走了一条从根结点到叶子结点的路径。

在MySQL索引中常用。

红黑树

红黑树性质：

  1. 每个结点非红即黑
  2. 根、叶结点黑
  3. 红结点两个儿子为黑
  4. 任一结点到其叶结点的每条路径包含相同数目的黑结点

红黑树这种数据结构应用十分广泛，在多种编程语言中被用作符号表的实现，如：
Java中的java.util.TreeMap,java.util.TreeSet；
C++ STL中的：map,multimap,multiset；
.NET中的：SortedDictionary,SortedSet 等。

键树

如果以树的多重链表表示键树，此时的键树又称Tire(取retrieve检索中间四个字符)树(字典树)。
字典树的基本性质：

  1. 根结点不包含字符，除根结点外每一个结点都只包含一个字符。
  2. 从根结点到某一结点，路径上经过的字符连接起来，为该结点对应的字符串。
  3. 每个结点的所有子结点包含的字符都不相同。

Trie的核心思想是空间换时间。利用字符串的公共前缀来降低查询时间的开销以达到提高效率的目的。典型应用是用于统计和排序大量的字符串（但不仅限于字符串），所以经常被搜索引擎系统用于文本词频统计。它的优点是：最大限度地减少无谓的字符串比较，查询效率比哈希表高。

哈希表

前面的各种结构的查找，都是需要比较，理想的情况是希望不经过任何比较，一次存取便能得到所查找的记录。那就必须在记录的存取位置和它的关键字之间建立一个确定的对应关系f，使每个关键字和结构中一个唯一的存储位置相对应。
哈希函数规则：通过某种转换关系，使关键字适度的分散到指定大小的顺序结构中，越分散以后查找的时间复杂度越小，空间复杂度更高。

算法流程：

  1. 构造哈希表：直接定址法H(key)=a*key+b、除留取余法H(key)=key%p
  2. 处理冲突：开放定址法Hi=(H(key)+di)%m di=1,2,…、链地址法
  3. 在哈希表的基础上执行哈希查找

哈希表是一个在时间和空间上做出权衡的经典例子。如果没有内存限制，那么可以直接将键作为数组的索引。那么所有的查找时间复杂度为O(1)；如果没有时间限制，那么我们可以使用无序数组并进行顺序查找，这样只需要很少的内存。哈希表使用了适度的时间和空间来在这两个极端之间找到了平衡。只需要调整哈希函数算法即可在时间和空间上做出取舍。

复杂度分析：对于无冲突的哈希表，查找复杂度为O(1)
