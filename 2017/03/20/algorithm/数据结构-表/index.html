数据结构-表

Posted on 2017-03-20 | In Note

时间复杂度：基本操作的重复执行次数
空间复杂度：算法所需存储空间的度量，按最坏情况来分析

线性表

线性结构的特点：第一个元素唯一、最后一个元素唯一、只有一个前驱、只有一个后继

线性表的顺序表示

线性表的顺序表示指的是用一组地址连续的存储单元依次存储线性表的数据元素。相当于C语言中的一个数组。
优点是可以随机存取任意元素，缺点是插入、删除操作要移动大量元素

线性表的链式表示

对于数据元素来说，除了存储其本身的信息之外，还需存储一个指示其直接后继的信息。这两个数据：数据域、指针域，构成一个结点，多个结点链接成一个链表。

随机访问操作：从头遍历到i
插入操作：修改指针即可。例如插入s到p结点后面s->next = p->next; p->next = s;
删除操作：修改指针，释放内存。例如删除p后面一个结点qq = p->next; p->next = q->next; free(q);
逆向创建链表：逐个插入结点。初始化L->next = NULL; 循环遍历p->next = L->next; L->next = p;

静态链表

可以用数组来表示链表，用游标（指示器cur）代指指针指示节点在数组中的相对位置。这种存储结构需要预先分配一个较大的空间，但是在线性表的插入和删除操作时不需要移动元素，仅需修改游标cur即可。

循环链表

表中最后一个节点的指针域指向头结点，整个链表形成一个环。
循环链表的操作和线性表基本一致，差别仅在于算法中的循环结束条件不是p或p->next是否为NULL，而是判断他们是否等于头指针。

双向链表

每个节点有一个指向直接后继的指针域，还有一个指向直接前驱的指针域。
双向链表也可以是一个循环链表，他会有两个环，当表为空的时候有d->next->prior = d = d->prior->next;

双向链表插入操作：修改四个指针。例如插入s到p前面s->prior = p->prior; p->prior->next = s; s->next = p; p->prior = s;
双向链表删除操作：修改两个指针，释放内存。例如删除p指针p->prior->next = p->next; p->next->prior = p->prior; free(p);

栈和队列

栈是限定仅在表尾进行插入或删除操作的线性表。表尾称为栈顶，表头称为栈底，后进先出。
base == NULL表示栈不存在，base == top表示栈为空，top - base == stack.size()

栈的应用举例

  • 数制转换：输入任意非负十进制数，输出对应的八进制数
    计算过程是从低位到高位顺序产生八进制数的各个位数，然后输出。而输出过程输出的数值顺序正好与实际的数字相反。因此将计算过程中产生的各个位数按顺序进栈，最后弹栈就是输出序列了

迷宫求解

计算机解迷宫时，通常用的是“穷举求解”的方法，即从入口出发，顺某一方向向前搜索，若能走通，则继续向前走；否则沿原路退回，换一个方向再继续探索，直至所有可能的通路都搜索到为止。为了保证在任何位置上都能沿原路退回，显然需要用一个后进先出的结构来保存从入口到当前位置的路径。

括号匹配的检测

如果遇到开括号压栈，如果遇到闭括号弹栈（先判断栈是否为空，防止出错），比较弹栈的数值是否与括号匹配，若匹配继续遍历，不匹配返回无法匹配。最后遍历完字符串如果栈为空，则括号完全匹配（还有一种情况是没有一个括号，这样也算匹配）

表达式求值

任何一个表达式都是由操作数operand、运算符operator和界限符delimiter组成的。首先存好每两个运算符的优先级大小，然后判断是否压栈即可

栈和递归

n阶汉罗塔问题：假设有3个分别命名为x、y和z的塔座，在塔座x上插有n个直径大小各不相同、从小到大编号为1到n的圆盘。现要求将x轴上的n个圆盘移动到塔座z上，并且按同样顺序放置，圆盘移动的时候不能将一个较大的圆盘压在较小的圆盘之上。

考虑递归：用y做辅助，我们要先将上面的n-1个圆盘移动到y上，然后就可以将最后一个移动到z上，然后将n-1个从y移动到z上。这就是一个递归的问题

队列

队列是一种先进先出FIFO的线性表。它只允许在表的一端进行插入，而在另一端删除元素。这和我们日常生活中的排队时一致的，最早进入队列的元素最早离开。在队列中，允许插入的一端叫做队尾(rear)，允许删除的一端则称为队头(front)。

链队列

队列可以使用链表存储，称为链队列。一个链队列需要两个分别指向队头和队尾的指针才能唯一确定。（空队列的判决条件为头指针和尾指针指向一个位置）
链队列的操作即为单链表的插入和删除操作的特殊情况，只是尚需修改尾指针或头指针。

  • 循环队列判断是否为空：front == rear
  • 循环队列判断是否为满：(rear+1) % MAXQSIZE = front
