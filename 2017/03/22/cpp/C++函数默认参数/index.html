C++函数默认参数

Posted on 2017-03-22 | In Note

默认参数是在函数声明时就给定的一个值，如果在调用函数时没有指定这一参数的值，编译器就会自动插入。
void f(int x, int = 0, float = 1.1);
这样看起来与函数重载十分相像，但是他们之间又有区别。

默认参数

默认参数的一个重要应用情况是在开始定义函数的时候用了一组参数，而使用了一段时间后发现要增加一些参数。通过把这些新增参数都作为默认的参数，就可以保证所有使用这一函数的客户代码不会受到影响。

使用默认参数需要注意以下几点：

默认参数在最右边

不可以在一个默认参数后面又跟一个非默认的参数。默认参数右边的参数都是默认参数。

默认参数在声明中设定

默认参数只可以在函数声明中设定一次，只有在没有函数声明的情况下才可以出现在函数定义中。
声明时参数可以没有标识符，这种参数被称为占位符，其目的在于以后可以修改函数定义而不需要修改所有的函数调用。当然，用一个有名字的参数也能够达到同样的目的，但如果定义的这个参数在函数体内没有使用它，多数编译器会给出警告信息，并认为犯了一条逻辑错误。

默认参数不能是局部变量

默认参数要在编译时确定，局部变量的位置和默认值在编译时无法确定。
默认参数的值一般可以是全局变量，全局常量或者一个函数。

类的默认构造函数

编译会报错，第二个有默认参数的构造函数与第一个系统自动生成的默认构造函数相同

函数重载

重载函数的思想是让我们用同名的函数，但是这些函数的参数列表应该不一样。编译器会修饰这些名字、范围和参数来产生内部名以供它和连接器使用。

上面两个在全局范围定义的函数，可能会产生类似于_print_char 和_print_float的内部名。

C++禁止用返回值重载。总是可以调用一个函数但忽略它的返回值，即调用了函数的副作用

函数重载和默认参数的选择

不能够把默认参数作为一个标志去决定执行函数的哪一块。
只要能够就应该把函数分解成多个重载的函数。
