C++语法总结

Posted on 2017-03-30 | In Note

C++的基础语法总结

关键字总结

inline

内联是在编译阶段插入代码，与预处理器宏相比，内联函数有类型检查而宏没有，预处理器不能访问类的成员变量而内联函数能够作为类的成员函数。

假如函数太复杂，编译器将不能执行内联。（循环、递归导致代码片段太大，无法插入）
加入要显式的或隐式的取函数地址，编译器也不能执行内联。（内联是在编译阶段插入代码，而不是跳转到函数中执行，所以没有函数地址）

与#define比起来，编写正确函数#define MAX(a,b) ((a) > (b) ? (a):(b))外面一层也要加，因为如果使用链式赋值int result = MAX(1,2)+2;加号运算符比三目运算符优先级高，如果没加最外层括号会先执行加号。
类中定义的函数自动成为内联函数（只声明不定义的不是内联）。如果想要在类的外面定义内联函数，要将内联函数和类的定义放到一个文件中。

static

  1. 变量维持值不变，如果变量在函数内，每次调用函数维持上一次的值，只初始化一次
  2. 变量内部属性，如果变量在函数外、模块内，在模块内所有函数都可访问，模块外不能
  3. 函数内部属性，函数只能被这一模块调用，模块外不能调用
  4. 类的static变量表示所有实例共享，某个实例更改了值其他所有实例都被更改
  5. 类的static函数为整个类所有，不接受this指针，只能访问类的static变量

const

  1. 在变量中：定义时必须赋值，除非extern。定义指针时要注意从右向左读
  2. 在函数中：修饰函数参数(const引用使用临时对象)、修饰函数返回值为重载提供参考
  3. 在类中修饰成员变量：const修饰的成员变量不能被修改，只能在初始化列表中赋值。
    static const变量必须在外部初始化
  4. 修饰成员函数：被const修饰类的成员函数，不能修改对象的成员变量，也不能调用类中的非const方法。const成员函数的this指针也是const的
  5. 修饰类对象、对象指针、对象引用：const修饰的类对象、指针、引用，其中的任何成员不能被修改，也不能调用任何非const成员函数。
    参见C++ const关键字使用

mutable

mutable也是为了突破const的限制而设置的。通常而言，const成员函数只能访问成员变量，不能修改成员，但是如果成员变量被mutable修饰了，则在const成员函数中可以修改该变量。
被mutable修饰的变量，将永远处于可变的状态，即使在一个const函数中。
mutable和const不能同时用于修饰成员变量。

volatile

用它声明的类型变量表示可以被某些编译器未知的因素更改，比如操作系统、硬件或者其它线程等。
当要求使用volatile 声明的变量的值的时候，系统总是重新从它所在的内存读取数据，即使它前面的指令刚刚从该处读取过数据。而且读取的数据立刻被寄存。volatile可以保证对特殊地址的稳定访问。
一个参数既可以是const还可以是volatile，一个例子是只读的状态寄存器。它是volatile因为它可能被意想不到地改变。它是const因为程序不应该试图去修改它。

cast

四种显式类型转换

  1. const_cast：去const或volatile属性
  2. static_cast：编译期间静态类型转换
  3. dynamic_cast：运行期间动态类型转化，出现在类到子类的转换，有虚机制（没有就用static_cast）
  4. reinterpret_cast：函数指针的转换，一般不用。
    参见C++ cast四种显式类型转换关键字

typedef

typedef用来定义一个数据类型的别名。与define不同，define是预编译时定义常量宏，typedef是编译时定义别名。define不受作用域约束。

typedef作用域struct时，在C++中使用struct对象可以省略struct关键字了。
typedef用于定义函数指针的别名。

struct

与class相比成员变量和成员函数默认是public的。
结构体存储时要保持对齐，结构体对齐规则：

  1. 普通数据成员：第一个数据成员放在offset为0的地方，以后每个数据成员存储的起始位置要从该成员大小的整数倍开始（比如int为4字节，则要从4的整数倍地址开始存储）。
  2. 结构体数据成员：如果一个结构里有某些结构体成员，则结构体成员要从其内部最大元素大小的整数倍地址开始存储。（struct a里存有struct b，b里有char，int，double等元素，那b应该从8的整数倍开始存储）。
  3. 结构体的总大小，必须是其内部最大元素占据字节的整数倍。

sizeof(X1)为16，sizeof(X2)为24，sizeof(Y1)为32，sizeof(Y2)为32
先分析X2：先根据规则1来补齐offset，b为int型offset为0，c为double型offset为4，补齐到8，a为char型offset为16。整个为17，最后根据规则3补齐总大小，要为8的倍数，补齐为24。所以sizeof(X)为24
再分析Y1：根据规则2，a为char型offset为0，B为X型offset为1，补齐到内部最大8，总共为8+24=32
分析Y2：b为X型offset为0，a为char型offset为24，根据规则3补齐到最大元素8，总共为24+8=32

在结构体struct中，enum和union如果不定义变量，其实并没有分配空间！

union

对于union来说，sizeof为union中大小最大的大小，对union赋值其他的值也会被修改。
union可以看作所有的成员起始地址相同，对union赋值就是对这整块内存赋值，所以根据大小端的不同，union中的存放值的内容会不同。

使用union判断系统是大端还是小端

返回false表示是小端，返回true表示是大端。
解释：小端表示高放高低放低，整个union占4个字节，赋值c.a=1，0x0001，如果是小端，1放在地地址，如果是大端，1放在高地址，此时c.b与c.a有相同的起始位置，所以如果c.b==1表示低地址放了1，表示小端。

new delete

new delete与malloc free的区别：

  1. new是运算符，malloc是库函数
  2. new分配内存、调用构造函数、返回正确地址；malloc只关心分配多少字节的内存。

指针

常量指针

str1和str2不相等，str3与str4相等
str都是地址，所以比较的是地址，str1和str2是新建一个长为4个字节的char数组，str1和str2都有自己的内存位置，所以不相等；str3和str4是指向一个字符串常量”abc”，这个字符串常量保存到常量表中，str3和str4都指向这个位置，所以相等

对上面进一步分析，str3和str4的值不能修改
str3[0] = b;会报错，因为”abc”存放在常量表中，实际上相当于const char* str = "abc";无法修改，要想修改不要这样初始化，最好使用malloc分配内存，然后strcpy，这样指针指向的内存存放在堆上，我们能够操作修改

数组指针

指针数组和数组指针：指针数组是一个数组，数组中是指针，数组指针是一个数组的指针。
指针函数和函数指针：指针函数时一个函数返回指针，函数指针是一个指向函数的指针。

[]的运算优先级比*高，所以要加括号

最后输出为p1的sizeof为4*10=40，p2的sizeof为4

同时注意数组指针的移位

*(a+1)为2,很明显
而&a+1表示a的地址向后移动一位，a是指向数组的指针，对a取址操作然后转换成指针，然后再向后移动一位，相当于指向了数组尾之后的一个位置。
*(ptr-1)表示 指向了数组尾之后的一个位置 然后向前移动一位，正好移动到数组尾

指针函数

右左原则：首先从最里面的圆括号(未定义的标识符)看起，然后往右看，再往左看。每当遇到圆括号时，就应该调转阅读方向。一旦解析完圆括号里面所有的东西，就跳出圆括号。重复这个过程直到整个声明解析完毕。

下面举例说明：

  1. int (*func)(int* p);
    首先找到那个未定义的标识符：func，他的外面有一对圆括号，而且左边是一个号，说明func是一个指针，然后跳出这个圆括号，先看右边，也是一个圆括号，说明(func)是一个函数，而func是指向这个函数的指针，即函数指针，这个函数有一个int*的形参，返回值类型是int
  2. int (*func)(int* p, int(*f)(int*));
    func被一对括号包围，且左边有一个号，说明func是一个指针，跳出括号先看右边，也是一个圆括号，那个func是一个指向函数的指针，函数中有两个参数，一个形参是int，另一个形参int (f)(int)，f也是一个函数指针，指向具有int*类型的形参，返回int
  3. int (*func[5])(int* p);
    func右边是一个[]，说明func是一个具有5个元素的数组，左边有一个，说明func的元素是指针（[]的优先级比高，所以修饰的是func[5]）。跳出这个括号先看右边，是一对圆括号，说明func数组的元素时函数类型的指针，他所指向的函数具有int类型的形参，返回值为int

C++11三个智能指针

unique_ptr不共享，不能使用copy符号，使用move转交占用、shared_ptr共享，加上一个引用计数、weak_ptr修饰shared_ptr，解决循环引用

引用

  1. 引用变量：相当于别名，不分配内存，必须被初始化，不能为NULL，不能更改，不能引数组
  2. 引用作为函数参数：不用复制参数对象
  3. 引用作为函数返回值：不用复制返回对象
  4. 在类中引用成员变量要在构造函数初始化列表中初始化

指针与引用区别

只能用指针不能用引用的情况：
指向数组、指向的内存可变、指向NULL

右值引用

&&，只能引用右值，直接将缓存值移过来，C++11 move语句将值变成右值

声明

声明(declaration)是向编译器介绍名字。他告诉编译器“这个函数/变量在某处可以找到，他的模样是怎么样的”
定义(definition)是“在这里创建变量/函数”，同时为这个名字分配存储空间。（定义函数编译器会生成代码，这些代码也需要占用一定的存储空间）
定义也可以是声明。如果在定义int x;之前编译器都没有发现标识符x，编译器就把这一标识符看成是声明并为他分配存储空间。

在变量定义前加extern关键字表示声明一个变量但不定义他：extern int a;。extern不能和static一起作用在一个变量上，static表示是有文件作用域的，而extern又表示是全局定义的，这样定义会冲突。
extern也可以用于函数声明，同正常的函数声明，是多余的：extern int fun(int length);

面向对象

面向对象和面向过程

面向对象：把数据和对数据的操作放在一起，封装成对象。对同类对象抽象出共性，即类，类通过一个简单的外部接口与外部发生联系。
面向过程是一种以事件为中心的开发方法，相当于一句话中的谓语。

出发点不同：对象，过程
层次逻辑关系不同：模拟客观世界，抽象成计算机世界
数据处理与程序程序控制方式不同：封装，直接处理

类

空类

空类默认6个成员函数：默认构造函数、析构函数、拷贝构造函数、赋值运算符重载函数、取址运算符重载函数&、const取址运算符重载函数

对空类求sizeof的大小是1，因为C++设计要求每个对象都有独特性，如果是空类的话C++编译器会为类添加一个标志符，所以大小为1

对AB两个类的对象求sizeof，大小都是1

对类对象求sizeof，先看是不是空类，然后看非静态成员变量，接着看继承关系，最后看虚表指针。
同时考虑上结构体对齐规则：
每个成员开始位置是自己的整数倍，最后大小要是最大的元素的整数倍

构造函数

构造函数没有返回值，只能抛出异常来判断是否成功构造对象，不过抛出异常之后要注意释放掉之前构造的一半的对象。

必须在构造函数初始化列表中初始化：
const数据成员、引用数据成员、没有默认构造函数的类成员、派生类初始化基类
初始化顺序为在类中定义顺序。
const static可以放在类中初始化，也可以放在外面初始化，外面初始化时不要加static

new操作符：获取一块内存空间、调用构造函数、返回正确的指针
operator new调用构造函数
placement new定位构造，在当前位置调用构造函数，不分配内存
new[] 在申请内存的时候在前面4个字节加上申请的对象个数，返回指针跳过这四个字节，delete[]的时候去前面查这4个字节就知道删多少了。

析构函数

析构函数不要抛出异常
纯虚虚构函数

拷贝构造函数

拷贝构造函数：A(const A& other);
深拷贝为当类有资源时复制资源，默认生成的为浅拷贝，只定义了新指针，没有复制资源

浅拷贝使得多个对象指向一块空间，然而当最后析构的时候，比如c先释放空间，而a，b却不知道还要释放空间便会产生重复释放同一内存的错误。
为此，我们可以对浅拷贝进行一个优化，例如在私有成员中加入一个int pCount来标记一块空间被几个对象占用，当只有一个对象占用如果进行析构便可释放空间，否则只对pCount–。

函数参数是const引用，引用时为了提高效率；const是不对other对象修改，同时能够接受const对象和非const对象（如果是非const参数的话，只能接受非const参数，C++不允许const转为非const）

赋值运算符

函数参数是const引用，分析同拷贝构造函数
返回值是对象的引用，返回的是这个对象的地址*this，为了能够链式执行a = b = c，如果不使用引用返回的是一个右值。（参考strcpy函数返回头指针，为了int len = strlen(strcpy(a,b));）

注意赋值运算符与拷贝构造函数的区别

赋值运算符由于要拷贝类的成员变量，所以不能是静态的（拷贝构造函数也是）
赋值运算符不能被继承，因为相较于基类，派生类往往要添加一些自己的数据成员和成员函数，如果允许派生类继承基类的赋值运算符重载函数，那么，在派生类不提供自己的赋值运算符重载函数时，就只能调用基类的，但基类版本只能处理基类的数据成员。

不能重载的操作符：
成员访问运算符.
成员指针访问运算符.*
域运算符::
长度运算符sizeof
条件运算符?:

public、protected、private继承
public不改，protected改基类的public为protected，private改基类所有为private。

在面向对象方法中一般是这样表述多态性的：向不同的对象发送同一个消息， 不同的对象在接收时会产生不同的行为（即方法）

运行期多态：虚函数
编译器多态：函数重载，函数覆盖，模板

虚函数

动态绑定（晚绑定）：运行时发生，静态绑定编译时发生

静态成员函数不能是虚函数，因为调用静态成员函数不需要实例
内联函数不能是虚函数，因为编译期间就展开了
构造函数不能是虚函数，因为运行构造函数前对象还没有生成，更谈不上动态类型了。构造函数要初始化虚表指针，而要调用虚函数的时候首先要知道虚表指针。

STL容器之间的比较

序列型容器

  1. vector：数组，连续空间，随机访问快，插入删除慢，内存不够重新分配两倍
  2. list：链表，不连续，随机访问要从头遍历，插入删除快，为每个元素分配空间
  3. deque：双端队列，小片连续，小片间用链表连接，内部有个map指针，所以知道类型，可以随机访问，支持随机插入删除。
    关联型容器：
  4. set：集合，红黑树实现（红黑树的结点就是每个元素），遍历的时候就排好序，元素唯一（相当于没有key的映射）
  5. map：一对一映射，红黑树实现（红黑树的节点是元素值和key点对，用key排序），key没有重复

iterator失效情况

iterator在指向的内存位置改变的时候会失效
insert操作只是更新结点指针的指向位置，没有改变内存位置，map、set的iterator不会失效
delete操作改变了内存，iterator失效

对于vector来说每一次插入删除，内存位置都可能改变，指针都可能失效(动态分配两倍)

C++11

右值引用

左值是有明确内存地址的值，不是左值的就是右值。
右值引用，是对临时对象的一种引用。

它是在初始化时完成引用的，但是右值引用不代表引用临时对象后就不能改变右值引用所引用对象的值，仍然可以在初始化后改变临时对象的值。

移动构造函数

在C++11之前，如果要将源对象的状态转移到目标对象只能通过复制。
而现在在某些情况下，我们没有必要复制对象——只需要移动它们。

move语句得到右值
正常情况下我们用对象A复制一个新对象B，对象A是个左值，对A对象做深拷贝得到B
现在我们使用移动构造函数，用右值对象A（使用move语句获得move(A)）得到一个新对象B，此时直接将A对象的资源的指针复制给新对象B，然后将A对象的资源指针置为nullptr，做了浅拷贝，而且还将对象A的指针置空。
右值对象A是临时值，会被系统自动清理。

nullptr

nullptr和任何指针类型以及类成员指针类型的空值之间可以发生隐式类型转换，同样也可以隐式转换为bool型（取值为false）。但是不存在到整形的隐式类型转换。

智能指针

unique_ptr

对象里存放的内容在整个程序中只能出现一次。不能有copy语句，只能是move语义的语句。

shared_ptr

可共享的指针，多个指针同时指向同一块内存。底层用引用计数方式实现，每增加一个指针指向内存，引用计数+1，减少一个指针指向内存，引用计数-1。当引用计数到0的时候，内存被清除。

weak_ptr

shared_ptr无法解决循环引用问题，为了避免循环引用，使用weak_ptr，若引用只引用而不增加或减少引用计数。如果一块内存被shared_ptr和weak_ptr同时引用，当所有的shared_ptr析构了之后，不管有没有weak_ptr引用该内存，内存都会被释放掉。
