C++编程思想 读书笔记

Posted on 2017-03-09 | In Note

2017/3/09 19:27 腾讯实习面试，问了许多C++基础知识，发现C++基础还是学的不扎实，先看一遍C++编程思想重学一遍C++。

对象导言

人们能解决问题的复杂性直接与抽象的类型和质量有关。
汇编语言是对底层机器的小幅度抽象。
“命令式”语言(Fortran、C)都是对汇编语言的抽象，但是他们的主要抽象仍然要求程序员按计算机的结构去思考，而不是按要解决的问题的结构去思考。

纯面向对象程序设计方法的五个特性：

  1. 万物皆对象
  2. 程序就是一组对象，对象之间通过发送消息互相通知做什么
  3. 每一个对象都有他自己的由其他对象构成的存储区
  4. 每个对象都有一个类型
  5. 一个特定类型的所有对象都能接收相同的消息

对象的创建和使用

语言的翻译过程

任何一种计算机语言都要从某种人们容易理解的形式（源代码）转换成计算机能执行的形式（机器指令）。通常，翻译器分为两类：解释器（interpreter）和编译器（compiler）

  1. 解释器
    解释器将源码转换成一些动作（由多组机器指令组成）并立即执行这些动作。
    解释器必须驻留内存以执行程序。
  2. 编译器
    编译器直接将源代码转换成汇编语言或机器指令。
    C语言使用分段编译，最后使用连接器(linker)将各段程序链接成一个完整的可执行程序。
    (python也支持分段编译)

编译型语言：先将程序源代码“翻译”成目标代码（机器语言），所以目标程序可以脱离其语言环境独立执行。运行效率较高，修改效率不高
解释型语言：解释器并不直接产生目标机器代码，而是产生易于执行的中间代码，然后由中间代码的解释程序执行。解释程序逐一将源程序的语句解释成可执行的解释程序执行。当语句出现错误时在开发期间就可以发现。运行效率较低，修改效率高。

声明和定义

声明(declaration)是向编译器介绍名字。他告诉编译器“这个函数/变量在某处可以找到，他的模样是怎么样的”
定义(definition)是“在这里创建变量/函数”，同时为这个名字分配存储空间。（定义函数编译器会生成代码，这些代码也需要占用一定的存储空间）
定义也可以是声明。如果在定义int x;之前编译器都没有发现标识符x，编译器就把这一标识符看成是声明并为他分配存储空间。

在变量定义前加extern关键字表示声明一个变量但不定义他。
extern int a;
extern也可以用于函数声明，同正常的函数声明，是多余的。
extern int fun(int length);

extern不能和static一起作用在一个变量上面。
如果已经把变量i定义为static，表示他是有文件作用域的（通过static），又要告诉编译器，i是全局定义的（通过extern），编译器会产生错误。

#include预处理器命令告诉预处理器打开指定的头文件并在#include语句所在的地方插入头文件。

库或程序中的每一个C++定义集被封装在一个名字空间中，如果其他的定义中有相同的名字，但他们在不同的名字空间，就不会产生冲突。

字符数组拼接：

如果两个加引号的字符数组邻接，并且他们之间没有标点，编译器就会把这些字符数组链接成单个字符数组。

连接

在一个执行程序中，标识符代表存放变量或被编译过的函数体的存储空间。连接用连接器所见的方式描述存储空间。

内部连接意味着只对正被编译的文件创建存储空间 。用内部连接，别的文件可以使用相同的标识符或全局变量。连接器不会发现冲突——也就是为每一个标识符创建单独的存储空间。内部连接由static关键字指定。
外部连接意味着为所有被编译过的文件创建一片单独的存储空间。一旦创建存储空间，连接器必须解决所有对这片存储空间的引用。

右值是能产生值的表达式。
左值是一个明确命名的变量（也就是说，应该有一个存储数据的物理空间）

类型安全连接

考虑如下例子

Def.cpp : 我能提供_f_int
Use.cpp : 我需要_f_char
会导致找不到f的外部链接。

但是在C中，编译器使用的内部名字不会加上函数参数，可能就是_f，这样的话会编译成功。
C++的这种使用名字修饰来提供一个安全网叫做类型安全链接。

函数重载与默认参数

函数重载

重载函数的思想是让我们用同名的函数，但是这些函数的参数列表应该不一样。编译器会修饰这些名字、范围和参数来产生内部名以供它和连接器使用。

上面两个在全局范围定义的函数，可能会产生类似于_print_char 和_print_float的内部名。

C++禁止用返回值重载。总是可以调用一个函数但忽略它的返回值，即调用了函数的副作用（side effect）

默认参数

默认参数是在函数声明时就给定的一个值，如果在调用函数时没有指定这一参数的值，编译器就会自动插入。
void f(int x, int = 0, float = 1.1);

默认参数的一个重要应用情况是在开始定义函数的时候用了一组参数，而使用了一段时间后发现要增加一些参数。通过把这些新增参数都作为默认的参数，就可以保证所有使用这一函数的客户代码不会受到影响。

不可以在一个默认参数后面又跟一个非默认的参数。
默认参数只能放在函数声明中。
声明时参数可以没有标识符，这种参数被称为占位符，其目的在于以后可以修改函数定义而不需要修改所有的函数调用。当然，用一个有名字的参数也能够达到同样的目的，但如果定义的这个参数在函数体内没有使用它，多数编译器会给出警告信息，并认为犯了一条逻辑错误。

函数重载和默认参数的选择

不能够把默认参数作为一个标志去决定执行函数的哪一块。
只要能够就应该把函数分解成多个重载的函数。

常量const

C++中的const默认为内部连接，也就是说，const仅在const被定义过的文件里才是可见的，而在连接时不能被其他编译单元看到。当定义一个const时，必须赋一个值给他，除非用extern做出了清楚的说明。(在C中默认const是外部链接的)

在C++中，一个const不必创建内存空间，而在C中，一个const总是需要创建一块内存空间。
如果取一个const的地址或者把他定义成extern，则会为该const创建内存空间。

聚合

const可以用于聚合，编译器不会真的把一个聚合保存到他的符号表中，所以必须分配内存。这种情况下，const意味着“不能改变的一块存储空间”。然而不能在编译期间使用它的值，应为编译器在编译期间不需要知道存储的内容。

函数参数和返回值

如果参数是按值传递的，指定参数是const的仅表示变量初值不会被函数改变，这个约定对客户来说是隐式的。

为了不使调用者混淆，采用第二种方式来在函数内部使用const限定参数。

对于内建类型来说，按值返回的是否是一个const是无关紧要的，所以按值返回一个内建类型时，应该去掉const以免让客户程序员混淆。
如果一个函数按值返回一个类对象为const时，那么这个函数的返回值不能是一个左值（即他不能被赋值，也不能被修改）

内联函数

在C保持效率的一种方法是使用宏，但是在C++中使用预处理器宏有两个问题。
第一个问题在C中也存在，宏只做扩展，会隐藏一些错误。
第二个问题是预处理器不允许访问类的成员数据，这意味这预处理器宏不能用作类的成员函数。

在C++中宏的概念是作为内联函数来实现的。
任何在类中定义的函数自动的成为内联函数，也可以在非类的函数前面加上inline关键字使之成为内联函数。
但是为了使之有效，必须使函数体和声明结合在一起，否则，编译器将它作为普通函数对待。

一般应该把内联定义放在头文件中。当编译器看到这个定义时，他把函数类型（函数名+返回值）和函数体放到符号表里。当使用函数时，编译器检查以确保调用是正确的且返回值被正确使用，然后将函数调用替换成函数体，因此消除了开销。内联代码的确占用空间，但加入函数娇小，这实际上比为了一个普通函数调用而产生的代码（参数压栈和执行CALL）占用的空间还少。

限制：有两种编译器不能执行内联的情况

  1. 函数太复杂。任何种类的循环都被人为太复杂而不扩展为内联函数，循环在函数里可能比调用要花费更多的时间。每次调用一个大的内联函数，整个函数体就被插入到函数调用的地方，容易使代码膨胀。
  2. 要显示或隐式地取函数地址。这时编译器必须为函数代码分配内存从而产生一个函数的地址。

引用和拷贝构造函数

C++中的引用

引用（reference）& 就像能自动地被编译器间接引用的常量型指针。
任何引用必须和存储单元联系。访问引用时，就是在访问那个存储单元。

如果从函数中返回一个引用，必须像从函数中返回一个指针一样对待。当函数返回时，无论引用关联的是什么都应该存在，否则，将不知道指向哪一个内存。

常量引用：

调用f(1)会产生编译期间错误，这是因为编译器必须首先建立一个引用，即编译器为一个int类型分派存储单元，同时将其初始化为1并为其产生一个地址和引用捆绑在一起。存储的内容必须是常量，对于所有的临时对象，必须同时假设他们是不可存取的。

类成员引用需要注意：

  1. 必须在初始化列表中指定
  2. 所指定的变量的生存期一定要长于该类的生存期。

在C/C++中，参数是从右向左进栈的。

运算符重载

运算符重载(operator overloading)只是一种”语法上的方便”(syntactic sugar)，也就是说他只是另一种函数调用的方式。

多态性和虚函数

多态性（polymorphism）提供了接口与具体实现之间的另一层隔离，从而将what与how分离开来。

虚函数

取一个对象的地址（指针或引用），并将其作为基类的地址来处理，这被称为向上类型转换(upcasting)
把函数体和函数调用相联系称为绑定(binding)。当绑定在程序运行之前（由编译器和连接器）完成时，这称作早绑定（C编译器只有这一种函数调用方式）
绑定根据对象的类型，发生在运行时，这种方法被称为动态绑定(dynamic binding)

为了引起动态绑定，C++要求在基类中声明这个函数时使用virtual关键字。

为了达到动态绑定的目的，编译器对每一个包含虚函数的类创建一个表(VTABLE)，在VTABLE中，编译器放置特定类的虚函数的地址。
在每个带有虚函数的类中，编译器秘密的放置一个指针，成为vpointer(VPTR)，指向这个对象的VTABLE。
当通过基类指针做虚函数调用时（也就是做多态调用时），编译器静态的插入能取得这个VPTR并在VTABLE表中查找函数地址的代码，这样就能调用正确的函数并引起动态绑定的发生。

抽象基类和纯虚函数

在设计时，常常希望基类仅仅作为其派生类的一个接口。可以在基类中加入至少一个纯虚函数(pure virtual function)，来使基类成为抽象类。
当继承一个抽象类时，必须实现所有的纯虚函数，否则继承出的类也将是一个抽象类。
virtual int fun() = 0;
后面=0，这样做相当于告诉编译器在VTABLE中为函数保留一个位置，但在这个特定位置中不放地址。只要一个函数在类中被声明成纯虚函数，则VTABLE就是不完全的。
当试图创建一个VTABLE不完全的类的对象时，编译器会发出错误信息，这样就保证了抽象类的纯洁性。

纯虚函数禁止对抽象类的函数以传值方式调用。通过抽象类，可以保证在向上类型转换期间总是使用指针或引用

虚函数和构造函数

当创建一个包含虚函数的对象时，必须初始化他的VPTR以指向对应的VTABLE。这必须在对虚函数进行任何调用之前完成。因为生成一个对象是构造函数的工作，所以设置VPTR也是构造函数的工作。
所有基类的构造函数总是在继承类构造函数中被调用。如果不在构造函数初始表达式表中显示的调用基类构造函数，他就调用默认构造函数。如果没有默认构造函数，编译器将报告错误。

如果在构造函数中调用了虚函数，被调用的这个函数只是这个函数的本地版本，也就是说虚机制在构造函数中不工作。

虚析构函数

构造函数是不能为虚函数的，但是析构函数常常必须是虚的。
构造函数有一项特殊工作，即一块一块的组合成一个对象。它首先调用基类构造函数，然后调用在继承顺序中的更晚派生的构造函数。类似地，析构函数也有一项特殊工作，即他必须拆卸属于某层次类的对象。为了做这些工作，编译器生成代码来调用所有的析构函数，但它必须按照与构造函数调用相反的顺序。

不把析构函数设置成虚函数是一个隐匿的错误，可能引起存储器泄露（关闭程序内存未释放）

在C++中纯虚析构函数必须提供一个函数体。
纯虚析构函数和非纯虚析构函数唯一的不同之处在于纯虚析构函数可以使得基类是抽象类，不能创建一个基类对象。（如果基类的任何其他函数是纯虚函数，也是具有相同的效果）

向下类型转换
