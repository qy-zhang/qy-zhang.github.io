C++继承的几种方式

Posted on 2017-03-28 | In Note

在编程中代码重用十分重要，如果希望更进一步，就不能仅仅使用拷贝代码，修改代码这种方式了。
在C++中我们可以通过创建新类来重用代码，这样能够直接使用别人已经创建好并经过调试的类。
创建新类的方式有两种：第一种方法很直接，在新类中创建已存在类的对象，这种方式称为组合。第二种方法就是继承。

public、protected、private继承

回顾一下三种访问权限：
public：可以被任意类对象访问
protected：只允许子类及本类的成员函数访问
private：只允许本类的成员函数访问

而对于继承来说，

  1. public继承不改变基类成员的访问权限
  2. protected继承将基类中public成员变为子类的protected成员，其他成员的访问权限不变
  3. private继承将基类中所有成员在子类中的访问权限变为private

所以不同继承方式只是在修改基类的public和protected成员访问权限，基类中的private成员在子类中无权访问。

几种继承的实例

所以总结起来就是：
public继承之后，类内部可以访问基类的public、protected，类对象只能访问基类的public
protected继承之后，类内部可以访问基类的public、protected，类对象访问不了任何基类的成员
private的继承之后，类内部可以访问基类的public、protected，类对象类对象访问不了任何基类的成员

private继承

如果要使用private继承，我们可以先考虑能否使用组合。
在private继承中，连基类的public和protected数据都不能访问，但是如果一定要访问，C++能够使用using baseClass::privateMember的形式访问。

多重继承和虚继承

C++允许多继承，但是多继承容易出现菱形继承问题和二义性问题。
菱形继承：B、C类都继承A类，D类多继承B类和C类。
二义性：继承的多个基类都有一个同名函数，当子类调用这个函数时不知道调用哪一个
C++使用虚拟继承来解决从不同途径继承来的同名的数据成员在内存中有不同的拷贝造成数据不一致问题。将共同基类设置成虚基类，这是从不同途径继承来的同名数据成员在内存中就只会有一份拷贝，同一个函数名也只有一个映射。

虚拟继承和普通继承的区别：
普通继承表示子类和基类是一个”is a”的关系，虚继承表示子类和基类是一个”has a”的关系。

虚基类在类对象中为每一个有关联的虚基类加上一个虚表指针vptr，改指针指向虚基类表。

不能被继承的类

利用友元和虚继承：
我们知道如果使用虚继承的话，最终派生类会直接调用虚基类的构造函数，我们通过这个特性，将虚基类的构造函数设置成private的。
具体来说就是虚基类A，直接派生类B，最终派生类C，我们想要做到的是让C无法继承B。利用的性质是C会直接调用A的构造函数，但是A的构造函数是private的，不能被访问。
但是对于B来说也不能访问，所以我们把B设置成A的友元，这样通过A和B的联合操作，使得没有任何类可以继承C。

类MakeSealed的构造函数和析构函数都是private的，但是由于SealedClass是他的友元类型，所以在SealedClass中调用MakeSealed的构造函数和析构函数都是合法的。
但是如果我们试图从SealedClass中继承一个类并创建一个实例的时候

由于类SealedClass是从类MakeSealed虚继承过来的，在调用Try的构造函数的时候，会跳过SealedClass而直接调用MakeSealed的构造函数，但是Try不是MakeSealed的友元类型，不能调用他的构造函数和析构函数，所以Try不能继承SealedClass
