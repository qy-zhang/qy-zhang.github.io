数据结构-排序

发表于 2017-03-21 | 分类于 Note

排序是将一个数据元素（或记录）的任意序列，重新排列成一个按关键字有序的序列。

假设Ki=Kj，且在排序前的序列中Ri领先于Rj。若在排序后的序列中Ri仍领先于Rj，则称所用的排序方法是稳定的。
举例说明：一个班的学生已经按照学号大小排好序了，现在要求按照年龄从小到大再排个序，如果年龄相同，必须按学号从小到大排序。

根据排序过程中涉及到的存储器不同，可以将排序方法分为两大类：内部、外部排序
内部排序：排序记录存放在随机存储器中进行的排序过程

排序性能分析：排序的基本操作为：比较两个关键字的大小和移动记录。

插入排序

直接插入排序

将一个记录插入到 已排好序的有序表中，从而得到一个新的有序表

先遍历比较大小，找到位置后插入，后面的元素相应往后移动一位。
遍历所有的数字，从第一个数开始一个一个数的插入，每次插入之前都是排好序的有序表。
时间复杂度为O(n^2)

折半插入

虽然减少了关键字之间的比较次数，但是记录的移动次数没有改变，因此折半插入排序的时间复杂度仍为O(n^2)

希尔排序

希尔排序(Shell’s sort)又称“缩小增量排序”，属于插入排序
从对直接插入排序的分析得知，其算法时间复杂度为O(n^2)，但是，若待排记录序列为“正序”时，其时间复杂度可提高至O(n)。由此可设想，若代拍记录序列按关键字“基本有序”，则直接插入排序效率就可以大大提高。

希尔排序基本思想：先将整个待排记录序列分割成若干子序列分别进行直接插入排序，待整个序列中的记录“基本有序”时，再对全体记录进行一次直接插入排序。

以n=10的一个数组49, 38, 65, 97, 26, 13, 27, 49, 55, 4为例

第一次gap=10/2=5，所以49和13一组、38和27一组、65和49一组、97和55一组、26和4一组
组内使用直接插入排序排好序之后结果为：
13, 27, 49, 55, 4, 49, 38, 65, 97, 26

第二次gap=5/2=2，所以13、49、4、38、97一组、27、55、49、65、26一组
组内使用直接插入排序排好序之后结果为：
4, 26, 13, 27, 38, 49, 49, 55, 97, 65

第三次gap=2/2=1，所有元素为一组，组内直接使用插入排序
4, 13, 26, 27, 38, 49, 49, 55, 65, 97

交换排序

起泡排序

起泡排序当初始序列为“正序”时只需要一趟排序，在排序过程中进行n-1次比较，为“逆序”需要n-1躺排序，在排序过程中进行n*(n-1)/2次比较。总时间复杂度为O(n^2)

理解冒泡排序：
i=0，内层循环j从0到n-1-0，每次比较a[j]和a[j+1]相邻两个位置的大小，将较大的换到后面去，这样i=0会将最大的数换到数组中最后一位上
i=1，内层循环j从0到n-1-1，这一次会将第二大的数换到数组的倒数第二位上
…
最后终止条件是i=n-1，j终止条件是n-1-i，是因为取了a[j+1]，要保证数组不越界，如果后面是比较a[j]和a[j-1]，则j的终止条件是n-i，i的终止条件是n

对冒泡排序还能进行优化，假如我们在一趟遍历过程中，发现没有一次发生了交换，说明这一段序列就是有序的，此时可以终止循环了

快速排序

快速排序是对起泡排序的一种改进

基本思想：通过一趟排序将待排记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小，则可分别对这两部分记录继续排序，以达到整个序列有序。

快排平均时间复杂度为O(nlogn)，不稳定

通常，快速排序被认为是所有同数量级O(nlogn)的排序方法中平均寻能最好的。但是，若初始记录序列按关键字有序或基本有序时，快速排序退化为起泡排序，时间复杂度为O(n^2)。为改进之，通常依“三者取中”的法则来选取枢纽记录，a[low]、a[high]、a[(low+high)/2]三个值的中间值来作为枢纽几率。

选择排序

基本思想：每一躺在n-i+1个记录中选取关键字最小的记录作为有序序列中第i个记录

简单选择排序

一趟简单选择排序操作：通过n-i次关键字之间的比较，从n-i-1个记录中选出关键字最小的记录，并和第i个记录交换之。
总的时间复杂度为O(n^2)

树形选择排序

又称锦标赛排序。首先对n个记录的关键字进行两两比较，然后再其中n/2个较小者之间再进行两两比较，如此重复，直至选出最小关键字记录为止。

时间复杂度为O(nlogn),但是这种排序方法使用的辅助存储空间较多，最大值进行了很多重复的比较。

堆排序

堆分为大根堆和小根堆，是完全二叉树。大根堆的要求是每个节点的值都不大于其父节点的值。
由于堆中每次都只能删除第0个数据，通过”取出第０个数据”再执行堆的删除操作、重建堆（实际的操作是将最后一个数据的值赋给根结点，然后再从根结点开始进行一次从上向下的调整。），然后再取，如此重复实现排序。
调整时先在左右儿子结点中找最小的，如果父结点比这个最小的子结点还小说明不需要调整了，反之将父结点和它交换后再考虑后面的结点。相当于从根结点将一个数据的“下沉”过程。

堆排序时间复杂度为O(nlogn)，不稳定，空间复杂度为O(1)

归并排序

归并排序，是创建在归并操作上的一种有效的排序算法该算法是采用分治法（Divide and Conquer）的一个非常典型的应用，且各层分治递归可以同时进行。
即先使每个子序列有序，再将两个已经排序的序列合并成一个序列的操作。若将两个有序表合并成一个有序表，称为二路归并。

首先介绍将两个有序数组合并为一个大的有序数组

归并排序，其基本思路就是将数组分成二组A，B，如果这二组组内的数据都是有序的，那么就可以很方便的将这二组数据进行排序。可以将A，B组各自再分成二组。依次类推，当分出来的小组只有一个数据时，可以认为这个小组组内已经达到了有序，然后再合并相邻的二个小组就可以了。这样通过先递归的分解数列，再合并数列就完成了归并排序。

时间复杂度为O(nlogn)，空间复杂度为O(n)，稳定

桶排序

桶排序（Bucket sort）或所谓的箱排序，是一个排序算法（属于分而治之）。
假设有一组长度为N的待排关键字序列K[1….n]。首先将这个序列划分成M个的子区间(桶) 。然后基于某种映射函数，将待排序列的关键字k映射到第i个桶中(即桶数组B的下标 i) ，那么该关键字k就作为B[i]中的元素。接着对每个桶B[i]中的所有元素进行比较排序(可以使用快排)。然后依次枚举输出B[0]….B[M]中的全部内容即是一个有序序列。
桶排序与归并排序、快速排序看起来好像很类似，都用到了分而治之的方法，而桶排序的重点在于通过关键函数将数据放到有序排列的桶中。
比如求0~1间的小数排序，可以分成10个桶，分别存入0~0.1, 0.1~0.2….，之后先在0.1的桶中排序，再归并的时间也是常数了，所以说桶数越多，所花的时间也越少。

基数排序

基数排序（Radix sort）是一种非比较型整数排序算法，其原理是将整数按位数切割成不同的数字，然后按每个位数分别比较。由于整数也可以表达字符串（比如名字或日期）和特定格式的浮点数，所以基数排序也不是只能使用于整数。

计数排序

首先介绍一种时间复杂度为O(n)，空间复杂度为O(1)的方法
相当于一个bitmap，为每个数字计数，最后一次遍历bitmap，将数据放到原先的数组中
