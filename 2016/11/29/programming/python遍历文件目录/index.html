Python遍历文件目录

Posted on 2016-11-29 | In Blog

项目中生成了一大批数据文件，想要遍历文件夹下的所有文件，然后按照文件序号来进行一些操作。
整体操作分两步，第一步遍历整个文件夹下的所有文件，将文件名存到一个list中；第二步对list排序，根据文件名中的序号升序排序。

遍历文件目录

利用os.listdir

os.listdir(path) return a list containing the names of the entries in the directory given by path. The list is in arbitrary order. It does not include the special entries ‘.’ and ‘..’ even if they are present in the directory.

该方法返回不带根目录的文件名或子目录名，输入参数为文件路径，输出参数为一个列表，包含列出的目录的文件名和子目录名。
可编写函数以备使用

利用os.walk

os.walk(top, topdown=True, onerror=None, followlinks=False)

Generate the file names in a directory tree by walking the tree either top-down or bottom-up. For each directory in the tree rooted at directory top (including top itself), it yields a 3-tuple (dirpath, dirnames, filenames).

返回一个包含3个项目的元组:当前目录名称,子目录名称,子文件名称

利用glob模块

glob.glob(pathname, *, recursive=False)

Return a possibly-empty list of path names that match pathname, which must be a string containing a path specification. pathname can be either absolute (like /usr/src/Python-1.5/Makefile) or relative (like ../../Tools//.gif), and can contain shell-style wildcards. Broken symlinks are included in the results (as in the shell).
If recursive is true, the pattern “**” will match any files and zero or more directories and subdirectories. If the pattern is followed by an os.sep, only directories and subdirectories match.

glob.glob(path)返回带目录的文件名.通配符和shell相似.path不能包含shell变量.

对文件名排序

列表的排序可以直接将文件名排序，但是如果直接使用sort函数排序会不符合我们的要求：排序的顺序不是按照数值大小顺序排序的。
比如：当文件名有 temp1、temp2、temp3、… temp10、temp20，排序会是 temp1、temp10、temp2、temp20、temp3 这种形式，因为这个排序是直接比较文件名字符串的大小的，考虑C语言中的strcmp的实现，两个指针指向两个字符串，如果指向的字符相同就往后走，不同返回相减的值，谁的字符大说明那个字符串大，所以就排成上述形式了。
但是我们希望的是前面从1到20这种，对文件名列表进行排序时对sort的key调整一下即可。取出1、2、3这些数字，对这些数字排序就好了。

此时对listdir函数返回的列表进行排序即可实现按照数字大小排序
files.sort(key = lambda x:int(x[4:]))
lambda函数返回的是x字符串的一个切片，从第5个元素到最后一个元素，就是得到temp..这后面的数字，然后sort的key是这个数字，就能够实现我们想要的效果了。
