STL仿函数学习

发表于 2017-05-27 | 分类于 Note

仿函数(functor)又被称为函数对象(function object)，是指一种具有函数性质的类对象。换句话说就是让一个类的使用看上去像是一个函数。
其实现就是类中实现一个operator()，在C++中，函数调用操作符是指左右小括弧() ，该操作符是可以重载的（复习一下不能重载的操作符 . .* :: ?: sizeof typeid）。这个类就有了类似函数的行为，就是一个仿函数类了。

总结一句就是：通过重载()运算符模拟函数行为的类。

要注意两点：

  1. 仿函数不是一个函数，而是一个类
  2. 仿函数重载了operator ()运算符，使得调用形式好像是在调用函数

C++返回值优化

发表于 2017-05-16 | 分类于 Note

返回值优化（Return value optimization，缩写为RVO）是C++的一项编译优化技术。即删除保持函数返回值的临时对象。这可能会省略两次复制构造函数，即使复制构造函数有副作用。

最近想要重构代码，首先考虑能从哪些方面优化。看完Effective C++之后感觉对引用有了新的认识，能用引用的地方尽量都不用值。函数传参数用const引用，返回值用const引用。（const是为了能够传递const参数，C++默认是允许非const向const传，不允许const向非const传）回顾一下引用参数和引用返回值参见之前博客

所以我就想，是不是函数返回值最好都换成一个引用参数传递进去。

答案是没有必要，至少有两点

  1. 从功能上来说，如果把函数都换成了void返回，把要返回的值作为了引用参数传递进去，这样不能使用链式操作了。
  2. 从性能上来说，直观上来看返回值会多次拷贝对象，但是实际上C++编译器会对返回值做优化，会省略拷贝析构的过程，性能上也没有问题了。（VS下要使用release模式）

所以在编写程序时还是正常的编写带返回值的函数即可。
