快速排序介绍

快速排序是基于分治的一种排序算法。
通过一趟排序将待排记录分割成独立的两部分，其中一部分记录的关键字均比另一部分记录的关键字小，则可分别对这两部分记录继续排序，以达到整个序列有序。

快速排序基本特征

时间复杂度：O(nlgn)。最快情况下O(N^2)
空间复杂度：O(nlgn)
不稳定

partition

快排最重要的是一趟快排的实现，最常用有以下两种写法：

填坑法

这种实现方式是从两头到中间，每次从一边拿一个数来填充另一边的坑。
开始记录下a[low]，作为pivotkey枢纽值

  1. 从最右边开始找到第一个比pivotkey小的数（如果是逆序的话就是大的数，调整判断符号即可），说明这个数应该被放到枢纽值左边去。
  2. 把上面的那个小的数放到a[low]这个位置
  3. 从最左边开始找到第一个比pivotkey大的数，说明这个数应该被放到枢纽值右边去。
  4. 把上面的那个大的数放到a[high]这个位置
    上面四步相当于放好了两个数的位置。
    while循环从两头走到中间，每次放好两个数，直到全部走完，有时候放好一个数之后low < high已经不满足了，就退出了。

举例说明算法流程
数组a{3,5,1,2,4}
开始pivotkey为3，loc为0，low为0，high为4
第一次while循环high减到3，a[3] = 2 < pivotkey，把a[high]放到a[low]处；low加到1，a[1] = 5 > pivotkey，把a[low]放到a[high]处。此时a为{2,5,1,5,4}
第二次while循环high减到2，a[2] = 1 < pivotkey，把a[high]放到a[low]处；low不再小于high，退出循环。此时a为{2,1,1,5,4}
最后将pivotkey枢纽值放到a[low]处，a变为{2,1,3,5,4}

遍历交换

举例说明算法流程
数组a{3,5,1,2,4}
开始pivotkey为3，loc为0，low为0，high为4
开始i为1，a[1] = 5 > pivotkey = 3，所以继续
接着i为2，a[2] = 1 < pivotkey，首先loc加1，表示比pivotkey小的元素有一个了，交换a[i]和a[loc]，把这个元素放到左边去，此时数组a为{3,1,5,2,4}
接着i为3，a[3] = 2 < pivotkey，loc加1，比pivotkey小的元素又多了一个，交换a[i]和a[loc]，把这个元素放到左边去，此时数组a为{3,1,2,5,4}
最后i为4，a[4] = 4 > pivotkey，结束if循环
交换枢纽元素和最终loc所在的位置，此时loc左边的所有元素都比a[loc]小了。数组变为{2,1,3,5,4}

从结果可见，两个写法等价。

当然，这种方法也可以选择最后一个元素作为枢纽。但是通过统计得出，三分区中这种方式的效率最高。就是说a[low]，a[high]，a[mid]这三个值取中间大的值作为枢纽最好。
