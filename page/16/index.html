操作系统笔记

发表于 2017-03-18 | 分类于 Note

计算机硬件

冯·诺伊曼结构

冯.诺依曼指出：程序只是一种（特殊）的数据，它可以像数据一样被处理，因此可以和数据一起被存储在同一个存储器中——这就是著名的冯.诺依曼原理。

冯·诺伊曼结构将程序指令和数据合并在一起，程序指令存储地址和数据存储地址指向同一个存储器的不同物理位置，因此程序指令和数据的宽度相同。
对比于哈佛结构：哈佛结构是一种将程序指令存储和数据存储分开的存储器结构。中央处理器首先到程序指令存储器中读取程序指令内容，解码后得到数据地址，再到相应的数据存储器中读取数据，并进行下一步的操作（通常是执行）。程序指令存储和数据存储分开，可以使指令和数据有不同的数据宽度。

使用冯·诺依曼结构的计算机，程序空间不封闭，期程序空间的数据在运行期理论上可以被修改，此外程序一旦跑飞也有可能运行到数据区。所以，相对于冯·诺依曼结构，哈佛结构更加适合于那些程序固化、任务相对简单的控制系统。

程序、进程、线程

几者区别

程序：一组指令的有序结合
进程：程序在某个数据集合上的一次运行活动（进程=程序+数据+PCB进程控制块）
线程：进程的一个实体，cpu调度和分派的基本单元
一个程序至少有一个进程，一个进程至少有一个线程
线程共享所属进程的所有资源、内存空间，进程之间相互独立、有独立的内存空间

  1. 进程是资源分配的基本单位，线程是CPU调度的基本单位。
  2. 进程之间资源独立（除了共享内存），同一个进程中的线程之间资源共享。所以进程切换消耗大，线程切换消耗小
  3. 通信、同步方式不一样

进程

多进程

多进程在Linux下使用fork函数创建。
子进程返回值为0，父进程返回值为子进程的进程ID。

四个进程，分别输出
进程一：1001, 1002、进程二：1001, 0、进程三：0，1003、进程四：0, 0

写时复制，子进程共享父进程数据段、栈和堆，访问权限改变为只读，如果父进程或者子进程需要修改这些数据，内核只为修改的那块内存拷贝一个副本。
子进程和父进程执行先后顺序根据内核进程调度算法而不同。

fork有以下两方面的用法

  1. 一个父进程希望复制自己，使父进程和子进程同时执行不同的代码段（使用if语句判断fork函数值返回是否为0来执行不同代码块）。这在网络服务器进程中最常见——父进程等待客户端的服务请求，当请求到达时，父进程调用fork产生子进程，子进程处理客户端的请求，而父进程接着等待客户端的请求。
  2. 一个进程要执行不同的程序。fork后面执行exec

一旦进程调用了exec函数，系统把代码段替换成新的程序的代码，废弃原有的数据段和堆栈段，唯一留下的是进程号。对系统而言，还是同一个进程，不过已经是另一个程序了。

进程三个基本状态

就绪、执行、阻塞
就绪→执行、执行→阻塞、阻塞→就绪、执行→就绪
本来是就绪、执行、阻塞三个的循环，但是多了一个执行到就绪状态，这个是当一个线程正在执行，但是时间片使用完了，所以从执行态转换成了就绪态。

进程调度算法

  1. 先来先服务：从就绪进程队列中选最先进入该队列的进程执行，直到执行完毕。
  2. 短作业优先：从后备队列中选择一个估计运行时间最短的作业，直到执行完毕。
  3. 优先权调度：从后备队列中选择优先权最高的作业，如果是非抢占式优先权调度会将该作业先执行完毕；如果是抢占式，如果执行过程中后备队列有优先权更高的作业就停止当前进程，切换进程转而执行更高优先权作业。
  4. 高响应比优先调度：用响应比来代表优先权，计算公式为 (等待时间+要求服务时间)/要求服务时间 ，然后将响应比作为优先权执行上面的优先权调度算法。
  5. 时间片轮转：先将CPU分配给队首进程，执行完一个时间片(几十ms)，时间片消耗完毕之后由计时器发出时间中断请求，调度程序接收终端信号停止进程的执行，将这个进程放到队尾。
  6. 多级反馈队列：设置多个就绪队列，各个队列优先级不同，优先级越高的队列时间片越短。新进程进来先到第一个进程队尾，如果进程能够在当前队列中的时间片执行完，就执行，如果不能执行完就放到下一个优先级的队列中。先将第一优先级队列执行完才执行第二优先级队列。

进程间通信

  1. 共享内存：共享内存是一段可以被多进程共享的物理内存，各个进程在使用之前，需要将这段物理内存映射到本进程的虚拟地址空间。多进程同时读写共享内存需要同步。
  2. 管道：半双工（数据只能在一个方向上流动）、进程需要有父子关系（有名管道FIFO可以用于任意两个进程间的通信）
  3. 信号量：相当于一个引用计数，PV操作，P表示获取资源，计数减1，V表示释放资源，计数加1。用来同步共享内存中的资源
  4. socket：网络间进程通信

线程

线程的基本概念

线程的5个基本状态：新建、就绪、执行、阻塞、死亡。就绪、执行、阻塞可以相互转换。
对比起进程来说，进程是就绪、执行、阻塞，这三个状态循环，其中只有执行->就绪这一条路可以倒着走，起因是时间片用完。

原子操作：则要么执行完所有步骤，要么一步也不执行，不可能只执行所有步骤的一个子集。
线程安全：多线程的程序运行结果是可预期的，多个线程同时调用该函数，允许出现互相影响的情况。
可重入：函数被多次调用但是结果具有可再现性。比线程安全要求更高，要求不使用、返回任何非常量的全局、静态变量，也不调用任何不可重入函数。

可重入函数必定是线程安全的，但是线程安全的函数不一定是可重入的。

多线程中栈是私有的，堆是公有的。

进程/线程死锁

  1. 互斥：一个资源每次只能被一个进程使用
  2. 请求和保持条件：进程获得一定资源后又对其他资源发出请求，又对自己获得的资源保持不防
  3. 不可剥夺：资源不可被剥夺，只能自己释放
  4. 循环等待：资源之间的循环链

线程同步和互斥

线程同步是指一个线程的执行依赖另一个线程的消息；互斥是指一个线程访问资源的时候另一个不能访问。

实现方式：

  1. 临界区Critical section：不是内核对象，只能用来一个进程内线程间的同步
  2. 互斥量Mutex：内核对象，能够确保线程拥有对单一资源的互斥访问权。运行速度慢
  3. 信号量Semaphore：内核对象，控制一个具有有限数量的用户资源
  4. 事件Event：通知线程有一些事件已发生

总结：
临界区效率最高，只能用于线程间互斥，不能用于进程间，所以不涉及用户态内核态切换，开销最小。
互斥量与临界区相比还能用于进程间互斥，信号量和事件也能用于进程间，可以用于同步。

用户模式下的方法有：原子操作（要么执行完，要么不执行）、临界区
内核模式下的方法有：事件、信号量、互斥量

银行家算法

允许进程动态地申请资源，但系统在进行资源分配之前，应先计算此次资源分配的安全性。若此次分配不会导致系统进入不安全状态，则将资源分配给进程；否则，令进程等待。

死锁是指两个或两个以上的进程在执行过程中，由于竞争资源或者由于彼此通信而造成的一种阻塞的现象，若无外力作用，它们都将无法推进下去。此时称系统处于死锁状态或系统产生了死锁，这些永远在互相等待的进程称为死锁进程。

操作系统按照银行家制定的规则为进程分配资源，当进程首次申请资源时，要测试该进程对资源的最大需求量，如果系统现存的资源可以满足它的最大需求量则按当前的申请量分配资源，否则就推迟分配。当进程在执行中继续申请资源时，先测试该进程已占用的资源数与本次申请的资源数之和是否超过了该进程对资源的最大需求量。若超过则拒绝分配资源，若没有超过则再测试系统现存的资源能否满足该进程尚需的最大资源量，若能满足则按当前的申请量分配资源，否则也要推迟分配。

哲学家就餐问题

5个哲学家（线程）围成一桌就餐，但是只有5只筷子（锁），一个人想要吃饭就必须要拥有左侧的筷子（锁1）和右侧的筷子（锁2）才能吃饭。每一个哲学家刚进桌前都持有了自己左侧的筷子，这样所有人只有一只筷子都无法就餐，所以就要想办法去拿右侧的筷子，而因为右侧的筷子被别人持有，所以无法拿到，这个时间就成了死锁状态。

哲学家就餐问题满足死锁的四个条件：

  1. 互斥：筷子不能同时使用
  2. 请求和保持条件：没有右边筷子不放左边筷子
  3. 不可剥夺：不能抢其他人的筷子
  4. 循环等待：坐成圆形，循环

解锁的条件：那就是在哲学家尝试去拿右侧筷子的时候，如果失败了，那么将自己左手边的筷子放下，此时这个哲学家左侧人就可以持有他原来左手边的筷子来就餐了。相当于破坏请求和保持这个条件。

内存管理

内存管理方式：

  1. 块式管理：将主存分为一大块一大块的，当所需的程序片段不在主存内就分配一块主存空间，把程序片段load入主存。易于管理、浪费内存空间
    进程A进来，向os申请了200的内存空间，于是os把0~199分配给A
    进程B进来，向os申请了5的内存空间，os把200~204分配给它
    进程C进来，向os申请了100的内存空间，os把205~304分配给它
    这个时候进程B运行完了，把200~204还给os
    但是很长时间以后，只要系统中的出现的进程的大小>5的话，200~204这段空间都不会被分配出去（只要A和C不退出）。
    过了一段更长的时间，内存中就会出现许许多多200~204这样不能被利用的碎片
  2. 页式管理：把主存分成一页一页的，可以适当消除内存的外零头，提高内存的利用率
  3. 段式管理：把主存分成一段一段的，每段空间比上述分页更小，这样做内存利用率更高，但是查询效率降低，一个程序片段可能会被分成几十段。
  4. 段页式管理：先将主存分成若干段，每个段又分成若干页，结合上述两种优点。段页式管理每取一次数据，要访问3次内存。

分页的地址空间是一维的（页面大小已知），拿到一个位置自己计算出来页号和页内位置
分段的地址空间是二维的，段号和段内地址

Cache替换算法

CPU处理数据块，容量小；内存容量大，转交给CPU处理的速度慢。Cache起到折中的作用，最有可能被使用到的数据先被内存调入Cache，CPU再从Cache中读取数据。CPU从Cache中读取到有用的数据成为“命中”。
Cache替换算法：先进先出(FIFO)算法、最近最少使用(LRU)算法、最近最不常用(LFU)算法

LRU首先淘汰最长时间未被使用的页面。
LFU淘汰一定时期被访问次数最少的页面。
若所需页面走向为2 1 2 1 2 3 4
若按LRU算法,应换页面1(1页面最久未被使用) 但按LFU算法应换页面3(十分钟内,页面3只使用了一次)
可见LRU关键是看页面最后一次被使用到发生调度的时间长短,
而LFU关键是看一定时间段内页面被使用的频率!

什么是中断？中断时CPU做什么工作？

中断是指在计算机执行期间，系统内发生任何非寻常的或非预期的急需处理事件，使得CPU暂时中断当前正在执行的程序而转去执行相应的事件处理程序。待处理完毕后又返回原来被中断处继续执行或调度新的进程执行的过程。
